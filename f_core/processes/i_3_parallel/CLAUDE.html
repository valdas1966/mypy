A. Purpose

  1. ProcessParallel[Item, Output] splits a list of items into chunks and executes them in parallel.
  2. Supports both thread-based (I/O-bound) and process-based (CPU-bound) execution.
  3. Error-tolerant: failed chunks produce None; errors are collected in errors.

_run() overrides ProcessIO 
def _run(self) -> list[Output | None]
  1. Resets errors, returns [] for empty input.
  2. Splits input into chunks via Group.to_groups().
  3. Selects ThreadPoolExecutor or ProcessPoolExecutor.
  4. Submits all chunks, collects results preserving order.
  5. Failed chunks set None in results and append to errors.

E. Usage Examples

Thread-Based (I/O-Bound) 
from f_core.processes.i_3_parallel import ProcessParallel

def square_chunk(chunk: list[int]) -> int:
    return sum(x * x for x in chunk)

proc = ProcessParallel(input=list(range(1, 13)),
                       func=square_chunk,
                       workers=3)
output = proc.run()
print(output)        # [14, 77, 194]
print(proc.elapsed)  # seconds
print(proc.errors)   # []

Error Tolerance 
from f_core.processes.i_3_parallel import ProcessParallel

def failing(chunk: list[int]) -> int:
    if 3 in chunk or 4 in chunk:
        raise ValueError('Simulated failure')
    return sum(chunk)

proc = ProcessParallel(input=list(range(1, 7)),
                       func=failing,
                       workers=3)
output = proc.run()
# Failed chunks → None, others succeed
for idx, chunk_data, exc in proc.errors:
    print(f'Chunk {idx}: {chunk_data} → {exc}')
