A. Purpose

  1. ProcessOutput[Output] is a generic process that produces typed output.
  2. Overrides run() to return the computed output instead of None.
  3. Level 1 process — single capability: output production.

def run(self) -> Output
  1. Executes _run_pre() → _run() → _run_post().
  2. Returns self._output after lifecycle completes.
  3. Key difference from ProcessBase.run() which returns None.
  4. Contract: subclasses must return the output from _run(). run() captures the return value into _output.
_output (private attribute) 
_output: Output | None = None
  1. Set by run() from _run()'s return value.
  2. Returned by run() after lifecycle completes.
  3. Initialized to None in constructor.

E. Usage Examples

Basic Subclass 
from f_core.processes.i_1_output import ProcessOutput

class Generator(ProcessOutput[list[int]]):
    def _run(self) -> list[int]:
        return [1, 2, 3, 4, 5]

result = Generator(name='Gen').run()
print(result)  # [1, 2, 3, 4, 5]
With String Output 
class Greeter(ProcessOutput[str]):
    def __init__(self, greeting: str):
        self._greeting = greeting
        super().__init__(name='Greeter')

    def _run(self) -> str:
        return f'Hello, {self._greeting}!'

message = Greeter(greeting='World').run()
print(message)  # "Hello, World!"
