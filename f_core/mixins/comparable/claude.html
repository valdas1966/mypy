Comparable

f_core / mixins / comparable
MIXIN

Purpose

Mixin for objects that support comparison operations (<, <=, >, >=). Extends Equatable and implements SupportsComparison by delegating all comparison operators to a single abstract key property.

All four operators are explicitly implemented for performance — no @total_ordering overhead. Each comparison is exactly 2 key accesses with no indirection. No type guards — comparing incompatible types raises AttributeError by design.

Usage Examples

Example 1 — Custom Subclass ▼ 
from f_core.mixins.comparable import Comparable


class Score(Comparable):
    def __init__(self, points: int) -> None:
        self.points = points

    @property
    def key(self) -> int:
        return self.points


s1 = Score(100)
s2 = Score(150)

s1 < s2   # True
s2 > s1   # True
s1 <= s1  # True
s1 == Score(100)  # True (from Equatable)

sorted([Score(50), Score(100), Score(75)])  # [50, 75, 100]
