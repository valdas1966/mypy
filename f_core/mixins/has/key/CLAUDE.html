HasKey

f_core/mixins/has/key/
MIXIN
Extends: Comparable, Hashable, Generic[Key]

Purpose

Mixin for objects that have a key. Provides a concrete key property, string representations, and inherits ordering from Comparable and hashing from Hashable. Generic over the Key type variable.

HasKey is the first concrete implementation of the abstract key property defined in Equatable. It stores the key as a private _key attribute and exposes it via a read-only property.

Usage Examples

Example 1: Basic HasKey Usage ▼ 
from f_core.mixins.has.key import HasKey

# Create instances with string keys
a = HasKey[str](key='A')
b = HasKey[str](key='B')
a2 = HasKey[str](key='A')

# String representation
str(a)          # 'A'
repr(a)         # '<HasKey: Key=A>'

# Equality (from Equatable)
a == a2         # True
a == b          # False

# Ordering (from Comparable)
a < b           # True
b >= a          # True

# Hashing (from Hashable)
hash(a) == hash(a2)  # True
{a, a2, b}           # two unique items
Example 2: Using the Factory ▼ 
from f_core.mixins.has.key import HasKey

# Factory creates pre-configured instances
a = HasKey.Factory.a()   # HasKey[str](key='A')
b = HasKey.Factory.b()   # HasKey[str](key='B')

str(a)          # 'A'
str(b)          # 'B'
repr(a)         # '<HasKey: Key=A>'

a < b           # True  (ordering by key)
a == b          # False
{a, b}          # usable in sets thanks to __hash__
