Usage & Usefulness

A) How to use it

Import and construct with a typed key, or use the factory methods:

from f_core.mixins.has.key import HasKey

obj = HasKey(key='my_key')
print(obj.key)    # 'my_key'
print(str(obj))   # string representation

# Factory shortcuts
a = HasKey.Factory.a()  # key='A'
b = HasKey.Factory.b()  # key='B'

B) When it's useful

Generic typed key holder used as a building block for comparable and hashable objects. Useful when domain objects need a single typed key that drives equality, ordering, and hashing, and you want that behavior composed via mixins.

C) When it's not useful

If your object has multiple keys or composite identity, a single-key mixin is insufficient. Also unnecessary if you do not need the Comparable/Hashable mixin chain.
