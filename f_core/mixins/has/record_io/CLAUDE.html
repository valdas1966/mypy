HasRecordIO

f_core/mixins/has/record_io/
MIXIN
Extends: HasName → Comparable, Hashable

Purpose

Mixin that gives objects separate input and output record containers. Composes two HasRecord instances behind read-only properties, providing a standard interface for objects that consume input data and produce output data (e.g., pipeline stages or processors).

No __init__.py exists for this module. The _factory.py exists but is incomplete.

Usage Examples

Example 1: Direct Instantiation ▼ 
from f_core.mixins.has.record_io.main import HasRecordIO

proc = HasRecordIO(name='MyProcess')
print(proc.name)                  # 'MyProcess'
print(proc.record_input.record)   # {}
print(proc.record_output.record)  # {}
Example 2: Subclassing for a Pipeline Stage ▼ 
from f_core.mixins.has.record_io.main import HasRecordIO

class Stage(HasRecordIO):
    def __init__(self, name: str) -> None:
        HasRecordIO.__init__(self, name=name)

stage = Stage(name='Transform')
print(stage.record_input)   # input HasRecord
print(stage.record_output)  # output HasRecord
