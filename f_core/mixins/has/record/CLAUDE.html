HasRecord

f_core/mixins/has/record/
MIXIN
Extends: HasName → Comparable, Hashable → Equatable

Purpose

Mixin for objects that can be converted to a flat record (dict). Each subclass declares a RECORD_SPEC mapping field names to getter lambdas. The mixin auto-merges specs from the entire MRO chain, producing a single record property that returns {field_name: value} (excluding None values). Also provides colored verbose printing.

Subclasses only need to define RECORD_SPEC with their own fields. The mixin walks the MRO and merges all specs automatically.

Usage Examples

Example 1: Using the Factory ▼ 
from f_core.mixins.has.record import HasRecord

a = HasRecord.Factory.a()   # A(HasRecord) with a=1
b = HasRecord.Factory.b()   # B(A) with a=1, b=2

print(a.record)             # {'name': 'A', 'a': 1}
print(b.record)             # {'name': 'B', 'a': 1, 'b': 2}
print(b.str_record())       # '[A=1] [B=2]'
print(b.header_record())    # ['name', 'a', 'b']
Example 2: MRO-based Spec Merging ▼ 
from f_core.mixins.has.record import HasRecord

class A(HasRecord):
    RECORD_SPEC = {'a': lambda o: o.a}
    def __init__(self):
        HasRecord.__init__(self, name='A')
        self.a = 1

class B(A):
    RECORD_SPEC = {'b': lambda o: o.b}
    def __init__(self):
        A.__init__(self)
        self.b = 2

b = B()
print(b.record)           # {'name': 'A', 'a': 1, 'b': 2}
print(b.header_record())  # ['name', 'a', 'b']
