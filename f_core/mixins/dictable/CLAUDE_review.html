Module Review Report

f_core/mixins/dictable/

Summary

Functional generic dictionary wrapper with a broad public API. Two correctness concerns: mutable default argument in __init__ and an unsafe __eq__ that will crash on non-Dictable comparisons. Several typing signatures need tightening. Design and modularity are clean.

Must-fix Issues (Correctness)

  • HIGH Mutable default argument — main.py:19: def __init__(self, data: dict[K, V] = None). The data or dict() guard on line 25 prevents the classic shared-mutable-default bug, so there is no runtime mutation issue. However, the type hint declares dict[K, V] but the default is None — this is a type error. Fix: 
    def __init__(self, data: dict[K, V] | None = None) -> None:
    self._data: dict[K, V] = data if data is not None else dict()
    Note: using data or dict() also discards an intentionally empty dict ({} is falsy). Using data if data is not None is more precise.
  • MED Unsafe __eq__ — main.py:118-124: self._data == other._data will raise AttributeError if other is not a Dictable (e.g., compared with a plain dict or None). Fix: 
    def __eq__(self, other: object) -> bool:
    if not isinstance(other, Dictable):
        return NotImplemented
    return self._data == other._data

Design Review

  • LOW Sizable.__bool__ conflict — Sizable provides __bool__ via bool(len(self)). This means an empty Dictable is falsy. This is correct Python semantics for a container, but worth noting: if a subclass also mixes in Validatable, the MRO determines which __bool__ wins. No action needed unless this conflict arises in practice.

Performance Notes

No change needed.

All methods are O(1) delegations to dict except keys(), values(), and items() which are O(n) due to list() conversion. This is intentional (returns a snapshot).

Typing Review

  • MED Constructor data parameter — main.py:19: Declared as dict[K, V] but defaults to None. Should be dict[K, V] | None = None.
  • MED get() return type — main.py:51: def get(self, key: K, default: V = None) -> V. When no default is provided, returns None, but the return type says V. Should be V | None. Also default: V = None should be default: V | None = None.
  • MED __eq__ parameter type — main.py:118: Should accept object per Python convention, not Dictable[K, V].
  • LOW Legacy Union import — main.py:1: Uses Union[dict[K, V], 'Dictable[K, V]'] in update(). Can use modern dict[K, V] | 'Dictable[K, V]' syntax (Python 3.10+, project uses 3.13+).
  • LOW Factory attribute type — main.py:17: Factory: type = None — missing | None. Project-wide pattern.
