9. Usage & Usefulness

A) How to use

Import and construct with an optional dict, then use as a dict-like object:

from f_core.mixins.dictable import Dictable

d = Dictable(data={'x': 10, 'y': 20})
d['z'] = 30
print(d.keys())    # ['x', 'y', 'z']
print(len(d))      # 3
print(bool(d))     # True

Use Dictable.Factory.abc() for quick test instances.

B) When useful

  1. When you need a typed, mixin-composable dict wrapper in the Sizable hierarchy.
  2. For domain objects that are fundamentally key-value stores but need to compose with other mixins.

C) When not useful

  1. If you only need a plain dict with no mixin composition — adds indirection without benefit.
  2. The wrapper is thin enough that it is not overengineered; every method delegates directly.

D) Better design

  1. No fundamental redesign needed. The class is a clean, minimal wrapper.
  2. Consider adding Printable to bases if the project convention expects it.
