f_core / protocols
4 PROTOCOLS

A. Purpose

  1. Structural typing contracts for equality, comparison, bounds, and rectangle conversion.
  2. All use typing.Protocol — no runtime inheritance required.
  3. Foundation layer consumed by f_core.mixins and f_ds.
Back to top

B. Public API

SupportsEquality (Protocol) 
def __eq__(self, other: object) -> bool
  1. Contract for equality checks.
  2. __ne__ omitted — Python derives it automatically.
SupportsComparison (SupportsEquality, Protocol) 
def __lt__(self, other: object) -> bool
  1. Contract for ordering operations (<, >, etc.).
  2. Extends SupportsEquality to include comparison.
  3. Concrete classes use @total_ordering to generate remaining operators.
SupportsBounds[T] (Protocol) 
@property
def bounds(self) -> Bounds[T]
  1. Contract for objects that have rectangular bounds.
  2. T constrained to int | float.
  3. @runtime_checkable — supports isinstance() checks.
RectLike (Protocol) 
def to_rect_coords(self) -> tuple[int, int, int, int]
  1. Contract for objects convertible to (x_min, y_min, x_max, y_max).
  2. Used by HasRowCol.is_within() for bounds checking.

F. Usage Examples

Type hint for equality 
from f_core.protocols import SupportsEquality


def are_equal(a: SupportsEquality, b: SupportsEquality) -> bool:
    return a == b


are_equal("hello", "hello")  # True
are_equal(1, 2)              # False
Type hint for ordering 
from f_core.protocols import SupportsComparison


def is_sorted(items: list[SupportsComparison]) -> bool:
    return all(a <= b for a, b in zip(items, items[1:]))


is_sorted([1, 2, 3])    # True
is_sorted(["c", "a"])   # False
Bounds checking with RectLike 
from f_core.protocols.rect_like import RectLike


def is_within(row: int, col: int, rect: RectLike) -> bool:
    x_min, y_min, x_max, y_max = rect.to_rect_coords()
    return x_min <= row <= x_max and y_min <= col <= y_max
