FileBase

Base class for all file types — file-system operations on a single file

What It Does

FileBase wraps a file path and provides common file-system operations. On creation, it auto-creates the file (and parent directories) if they don't exist. This guarantees that every FileBase instance points to a real file.

It serves as the foundation for all specialized file types in the framework.

API Reference

Constructor
__init__(path) None Store path, create parent dirs and empty file if not exists
Properties
.path Path Full file path (pathlib.Path object)
.name str File name with extension (e.g., main.tex)
.stem str File name without extension (e.g., main)
.suffix str File extension (e.g., .tex)
.size int File size in bytes
Methods
exists() bool True if the file exists on disk
delete() None Delete the file; silently ignores if already gone

Usage Examples

# Create or open a file
from f_psl.file.i_0_base import FileBase

# Auto-creates if not exists
f = FileBase(path='data/output.tmp')

# Inspect
print(f.name) # 'output.tmp'
print(f.stem) # 'output'
print(f.suffix) # '.tmp'
print(f.size) # 0
print(f.exists()) # True

# Clean up
f.delete()
print(f.exists()) # False
f.delete() # no error

Design principle: FileBase handles file-system concerns only — existence, path, size, deletion. Content operations (read/write text, parse structure) belong to subclasses like FileTxt and FileTex.
