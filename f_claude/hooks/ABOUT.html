? What Are Hooks?

Hooks are user-defined shell commands that fire at specific lifecycle moments — deterministically, every time, no LLM judgment involved.

Configured in settings.json, hooks bridge Claude Code and external systems. They can block tool calls before execution, modify inputs on the fly, validate state after actions, inject context into the conversation, and integrate with external services via HTTP.

Unlike skills (instructions Claude follows) or CLAUDE.md (static docs Claude reads), hooks are deterministic code that always runs, always behaves the same way, and doesn't rely on Claude's decision-making.

# Lifecycle Events (~20 events)

SessionStart Session

Session begins or resumes. Can write env vars to $CLAUDE_ENV_FILE.
Can Block matcher: source

SessionEnd Session

Session terminates (clear, logout, exit).
Observe Only

UserPromptSubmit Input

User submits a prompt, before Claude processes it. Stdout adds context.
Can Block

PreToolUse Tool

Before any tool executes. Can block, modify input, or change permissions.
Can Block matcher: tool_name

PostToolUse Tool

After tool succeeds. Action already completed — observe and react.
Observe Only matcher: tool_name

PostToolUseFailure Tool

After tool fails. Observe error, log, or alert.
Observe Only matcher: tool_name

PermissionRequest Tool

Permission dialog about to show. Auto-allow, auto-deny, or modify.
Can Block matcher: tool_name

Stop Agent

Claude finishes responding. Exit 2 forces Claude to keep working.
Can Block

SubagentStop Agent

Subagent finishes. Prevent premature completion.
Can Block

Claude sends notification (permission, idle, auth). Fire desktop alerts.
Observe Only matcher: notification_type

Before/after context compaction. Re-inject critical context.

Settings or skills files change. Audit or block config modifications.

</> Four Handler Types

command

Run a shell script. Reads JSON from stdin, writes to stdout/stderr.
command — script path
timeout — 0-3600s (default 600)
async — background fire-and-forget
statusMessage — shown during run

http

POST JSON to a URL. Response body parsed as hook output.
url — endpoint
headers — with $VAR interpolation
allowedEnvVars — whitelist
timeout — 0-300s (default 30)

prompt

Ask a lightweight LLM for a decision. Returns {ok, reason}.
prompt — question text
model — default Haiku
timeout — 0-300s (default 30)

agent

Spawn a subagent with tool access (up to 50 turns). Returns {ok, reason}.
prompt — task description
model — default Haiku
timeout — 0-600s (default 60)

! Exit Code Contract

0
Success
Action proceeds. Stdout parsed as JSON (if present) or added as context.
2
Block
Action blocked. Stderr fed to Claude as the reason. Only works on blocking events.
*
Non-Blocking Error
Action proceeds anyway. Error logged in verbose mode. Hook failure is not fatal.

{} Configuration Format

// settings.json (project, local, or user)
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash|Edit|Write",  // regex on tool_name
        "hooks": [
          {
            "type": "command",
            "command": "/path/to/validate.sh",
            "timeout": 30
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Edit",
        "hooks": [
          {
            "type": "command",
            "command": "prettier --write $(jq -r '.tool_input.file_path')"
          }
        ]
      }
    ],
    "Stop": [
      {
        "matcher": "",  // empty = match all
        "hooks": [
          {
            "type": "agent",
            "prompt": "Verify all tests pass before stopping"
          }
        ]
      }
    ]
  }
}

~ What Hooks Can Do

Block Tool Calls
PreToolUse • exit 2
Prevent rm -rf, database DROP, or writes to protected files before they execute.
Modify Tool Input
PreToolUse • updatedInput
Silently replace grep with rg, or rewrite unsafe commands on the fly.
Auto-Format Code
PostToolUse • Edit matcher
Run Prettier/Black after every file edit. Consistent style, zero effort.
Inject Context
SessionStart / UserPromptSubmit
Print git log, environment info, or key docs to stdout — added to Claude's context.
Audit Logging
PreToolUse • http type
POST every tool call to a central compliance service. Full action trail.
Auto-Grant Permissions
PermissionRequest
Auto-allow safe operations, auto-deny dangerous ones. No manual clicking.
Persist Env Vars
SessionStart • $CLAUDE_ENV_FILE
Write export NODE_ENV=prod once; every future Bash command inherits it.
Prevent Premature Stop
Stop • agent hook
Spawn a subagent to verify tests pass before Claude is allowed to stop working.

+/- Pros & Cons

+ Pros

1
Deterministic Enforcement Always runs. No LLM judgment. Exit 2 = blocked, period. Rules that can't be talked around.
2
Pre-Execution Blocking Stop dangerous commands before they execute. Not after — before.
3
Input Modification Silently rewrite tool inputs. Replace grep with rg, add flags, sanitize paths.
4
Four Handler Types Shell commands, HTTP endpoints, LLM prompts, or full subagents. Right tool for each job.
5
Granular Event Coverage ~20 lifecycle events. Hook into tools, sessions, agents, compaction, config changes, MCP.
6
Regex Matchers Filter by tool name, event source, agent type. Only fire when relevant.
7
Team Shareable Project-level hooks in .claude/settings.json commit to git. Everyone gets them.
8
Async Background Mode Fire-and-forget for deployments, logging, or long-running side effects.

− Cons

1
No Undo Capability PostToolUse can't reverse completed actions. Must use PreToolUse to prevent.
2
JSON-Only Communication No binary data, no streaming. Text/JSON in, text/JSON out.
3
Stop Hook Infinite Loops A Stop hook that keeps blocking causes repeated re-runs. Must check stop_hook_active.
4
Shell Profile Interference echo in ~/.bashrc breaks JSON parsing. Must guard with interactive check.
5
No Hook-to-Hook Communication Hooks are independent. No shared state unless you use external files/services.
6
All-or-Nothing Disable disableAllHooks: true kills everything. Can't selectively disable one hook.
7
Async Hooks Can't Decide Background hooks can't block or modify input. Only sync hooks control flow.
8
Case-Sensitive Matchers "bash" != "Bash". Exact casing required in regex patterns.
