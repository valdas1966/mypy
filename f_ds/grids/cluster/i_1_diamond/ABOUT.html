What is a Diamond Cluster?

Every cell within steps Manhattan moves of a center — a BFS ball, shaped like a diamond, that walls can dent.

The Idea

💧 Drop a stone in a pond. The ripple spreads one ring per tick. After steps ticks, every cell the ripple has reached is in the cluster. On a 4-direction grid the ripple isn't a circle — it's a diamond. And if a rock (a wall) blocks the water, the ripple flows around it, never through.

Why a diamond? Because the grid only lets you step up/down/left/right. "Distance" is then Manhattan distance — |Δrow| + |Δcol| — and the set of points at distance ≤ s is a 45°-rotated square: a diamond.

The diamond grows one ring per step

steps=0 → 1 cell
steps=1 → 5 cells
steps=2 → 13 cells
steps=3 → 25 cells

The counts follow a clean closed form: |D(s)| = 2s² + 2s + 1 → 1, 5, 13, 25, 41, 61, … — the centered square numbers.

Four rules of the diamond

  1. Metric is Manhattan. Membership = |Δr|+|Δc| ≤ steps, because the grid is 4-connected.
  2. Built by BFS, not by formula. It floods outward through real neighbors — so it returns the connected component containing the center.
  3. Walls dent it. A wall that bisects the geometric ball truncates the cluster to just the center's side. Geometry ≤ BFS.
  4. The center is the representative. The seed cell is the cluster's distinguished member — both center and representative.

What a wall does

open grid → 13
wall column → 10 (right lobe unreachable)

How It's Built

ClusterDiamond is one concrete shape under the abstract ClusterGrid. It inherits all the plumbing and adds two things: a BFS _build and a Hashable identity.

ClusterGrid abstract # root: name, members, representative, map, cells (light) └── ClusterDiamond(ClusterGrid, Hashable) concrete │ ├─ _build() # BFS ripple to depth=steps ├─ center == representative └─ key = (map, center.key, steps) # eq + hash 
def _build(self, grid: GridMap) -> list[CellMap]:
    """BFS from center to depth ≤ steps; walls skipped via grid.neighbors."""
    visited = {self._center.key}
    frontier = [self._center]
    cells    = [self._center]
    for _ in range(self._steps):           # one ring per step
        next_frontier = []
        for cell in frontier:
            for nbr in grid.neighbors(cell=cell):  # walls already filtered
                if nbr.key in visited: continue
                visited.add(nbr.key); cells.append(nbr); next_frontier.append(nbr)
        frontier = next_frontier
    return cells

Identity & Construction

🔑 Hashable by spec, not by contents

Two diamonds are equal iff their (map, center.key, steps) match — no member-list comparison. So a diamond drops straight into a set or dict key.

c1 = ...at_center(g, g[5][5], steps=2)
c2 = ...at_center(g, g[5][5], steps=2)
c1 == c2          # True
{c1, c2} == {c1}  # True

🏭 Factory holds the recipes

The constructor takes the raw grid, center, steps. Construction variants live in the Factory, keeping the ctor single-purpose.

This shape only makes sense against its base. The base is the noun ("a cluster of cells"); the diamond is the adjective made real ("the Manhattan-ball one").

Where Diamonds Show Up

🧭

A* heuristic

On a 4-connected grid, Manhattan distance is the classic admissible heuristic. A diamond is exactly "all cells with h ≤ steps."

Fun Facts & History

🚕 Why "Manhattan"

In a city laid out on a grid, a taxi can't cut diagonally through buildings — it drives the blocks. So |Δx|+|Δy| is the real travel distance. Hermann Minkowski formalized it (1900s) as the L¹ norm; the diamond is its unit ball.

🔷 A circle that thinks it's a square

Every shape here is a "ball" — the set of points within radius r. Change the metric and the ball changes shape: L¹ → diamond, L∞ → square, L² → circle. Same idea, three silhouettes.

🌊 BFS = Dijkstra with equal weights

The ring-by-ring flood is breadth-first search, which is just Dijkstra when every step costs 1. The frontier is the diamond's perimeter at each radius.

🔢 Centered square numbers

2s²+2s+1 (1, 5, 13, 25, 41…) are the centered square numbers — the count of points in an L¹ ball, and the cells of a diamond board in Aztec-diamond combinatorics.
