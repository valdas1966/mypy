Heuristic Computations: kA*inc vs kA*agg

A deep analysis of who pays what, when, and why
Based on: SoCS 2026 Paper #5875 — Sections 3.1 & 3.2
Section 1

The Core Question

Both algorithms expand the exact same set of nodes (Theorem 1). So the only difference is: how many times does each algorithm call the heuristic function hi(n)?

kA*agg (Aggregative)

Eager evaluator

F(n) = g(n) + mini∈A hi(n)

When a node is generated, compute hi(n) for ALL active goals to calculate the min. Cache the values — re-keying after goal removal is free.

Pays upfront for all goals
kA*inc (Incremental)

Lazy evaluator

fi(n) = g(n) + hi(n)

When a node is generated, compute hi(n) for only the current goal. When transitioning to the next goal, re-key nodes still in OPEN (one new h per node).

Pays only when needed

The Proof in One Sentence

Every heuristic evaluation performed by kA*inc (generation or re-key) has a corresponding evaluation in kA*agg (from the same node's eager batch). But kA*agg may compute evaluations that kA*inc never needs — for nodes expanded before those goals become active. The mapping is injective from inc to agg, never surjective. Therefore: Totalinc ≤ Totalagg, always.

kA*agg never has fewer heuristic evaluations than kA*inc

Sharp Summary

  1. 1
    Eager vs Lazy. That's the entire story. kA*agg computes heuristics for all active goals the moment a node is born. kA*inc computes one, then pays more only if the node is still sitting in OPEN when goals rotate. Lazy never overpays. Eager always can.
  2. 2
    kA*inc ≤ kA*agg — always, in total heuristic evaluations. There is no scenario where aggregative wins. The best aggregative can do is tie.
  3. 3
    They tie only when zero nodes are expanded before all goals are found — every generated node stays in OPEN until the end. This requires the search to generate nodes without ever consuming them. Practically impossible.
  4. 4
    Maximum savings = factor k. Achieved when OPEN is empty at every goal transition (all generated nodes are expanded within their phase). On grids with distant goals, this is nearly the case.
  5. 5
    The savings scale with two variables: (a) k — more goals = more waste by eager evaluation, (b) O/N — smaller OPEN-to-total ratio = more nodes escape re-keying. Formula: savings = k / (1 + (k−1) × O/N).
  6. 6
    On grids (paper's domain), OPEN = perimeter = O(√N), so O/N → 0. Savings approach the full factor k. On trees, OPEN ≈ N, savings collapse to 1x. Graph topology determines how much lazy wins.
  7. 7
    One sentence: Every heuristic call in kA*inc has a matching call in kA*agg, but not vice versa. The unmatched calls in kA*agg are heuristics computed for goals that a node will never be relevant to — because it gets expanded before those goals become the search target. That's pure waste.
