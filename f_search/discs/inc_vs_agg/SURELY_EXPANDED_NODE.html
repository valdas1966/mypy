The Surely Expanded Node

The irreducible core of search work — nodes that every optimal algorithm must expand. No shortcut, no trick, no aggregation function can avoid them.
Definition 11 — "Heuristic Search for One-to-Many Shortest Path Queries" (Stern, Goldenberg, Saffidine, Felner)
TL;DR
A surely expanded node has f(n) < optimal cost — every optimal algorithm must expand it. It defines the minimum work any search can do.

Why are (0,0) and (0,1) surely expanded?
Their f-values are 3 — far below d(s,t) = 7. A* pops the lowest-f node first. These nodes will be popped and expanded before A* can possibly reach the goal. No algorithm can skip them and still guarantee optimality.
Why is the bottom row surplus?
Every cell in row 3 has f = 9 > 7. By the time A* would consider expanding them, it has already found the optimal path. Expanding them is pure waste — they contribute nothing to proving optimality.

★ Formal Definition

Surely Expanded Node DEF 11

A node n is surely expanded w.r.t. heuristic h and goal t if there exists a path from s to n where all nodes n' on that path (except s, including n) satisfy:

d(s, n') + h(n') < d(s, t)
Simplified (consistent heuristics):  n is surely expanded  ⟺  d(s, n) + h(n) < d(s, t)

Any correct best-first search algorithm must expand these nodes to guarantee optimality — there is no way to certify the optimal cost without looking at every node that appears cheaper.

Surplus Node (contrast) DEF 11
d(s, n) + h(n) > d(s, t)   →   n is surplus (can be skipped)

The best possible cost through n exceeds the optimal cost. No algorithm needs to expand it. See SURPLUS_NODE.html for full details.

◆ OMSPP: What Happens with Multiple Goals?

Extension to k × A* DEF 12
SURELY EXPANDED
SE for at least 1 of k goals
SURPLUS
Surplus for ALL k goals
Key asymmetry: One goal can force a node to be surely expanded. But ALL goals must agree a node is useless before it counts as surplus. Adding more goals makes it harder for nodes to be surplus.

✓ Key Takeaways

1
Surely expanded = irreducible minimum. Every optimal algorithm must expand these nodes. They define the theoretical lower bound on node expansions.
2
Simple test with consistent h: If d(s,n) + h(n) < d(s,t), the node is surely expanded. Compare f-value to optimal cost.
3
OMSPP: one goal is enough. A node is surely expanded for k×A* if it's surely expanded for at least one of the k goals.
4
Phi = min is optimal. It's the only admissible aggregation that expands exactly surely expanded + borderline, with zero surplus waste.
