The Surplus Node

A visual guide to nodes that no optimal search algorithm ever needs to expand
Definition 11 — "Heuristic Search for One-to-Many Shortest Path Queries" (Stern, Goldenberg, Saffidine, Felner)
TL;DR
A surplus node has f(n) > optimal cost — expanding it is pure waste. Only Φ=min guarantees zero surplus expansions.

◆ Formal Definitions

◆ Surplus Node Def 11
n is surplus  ⇔  for every path s→n, ∃ node n' ≠ s :   d(s, n') > d(s, t)
Plain English: You cannot reach node n without passing through a node that already costs more than the optimal solution. Expanding n is guaranteed waste — it can never be part of, or lead to, an optimal path.
With consistent heuristics:   n is surplus  ⇔  d(s, n) + h(n) > d(s, t)
◆ Surely Expanded Node (contrast) Def 11
n is surely expanded  ⇔  ∃ path s→n where ∀ n' on path (n' ≠ s):   d(s, n') + h(n') < d(s, t)
Plain English: There is at least one route to n where every intermediate node looks strictly cheaper than the optimal cost. Any correct A* must expand these — they are unavoidable.
◆ Extension to OMSPP (k goals) Def 12
Category Condition for k×A*
Surely Expanded Surely expanded for at least 1 of the k goals
Surplus Surplus for ALL k goals — irrelevant to every single goal
Key asymmetry: Being useful to any one goal saves a node from surplus status. It must be useless to all k goals to be surplus.

Φ Impact of Aggregation Function (Φ)

● Φ = min

  1. Never expands any surplus node of k×A*
  2. The only admissible Φ with this guarantee
  3. Expands the same set of nodes as k×A* (Corollary 2)
Theorem 7 — Surplus-free

● Φ ≠ min (projection, mean, median)

  1. Can expand surplus nodes of k×A*
  2. Surplus expansions = wasted computation
  3. Projection is most affected (single-goal focus can drift into irrelevant regions)
Theorem 7 — Surplus possible

Key Takeaways

  1. Surplus nodes have f(n) > d(s, t) — expanding them is provably useless.
  2. For OMSPP, a node is surplus only if surplus for ALL k goals.
  3. Φ=min is the only aggregation that guarantees zero surplus expansions.
  4. Surplus count is a direct measure of wasted computation in kA*.
