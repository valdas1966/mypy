ProblemOMSPP

f_search/problems/i_2_omspp/

A. Purpose

  1. Defines the One-to-Many Shortest Path Problem: single start, multiple goals on a grid.
  2. Composes ProblemSearch, HasStart, and HasGoals mixins; only adds to_spps().
  3. Constructor accepts Grid (not Grid | str); inherits __getstate__/load_grid pickle support from ProblemSearch.

E. Usage Examples

Example 1: Factory construction 
from f_search.problems import ProblemOMSPP

# Without obstacles (4x4 grid, start=(0,0), 2 goals)
problem = ProblemOMSPP.Factory.without_obstacles()
print(problem.start)   # State at (0,0)
print(problem.goals)   # [State at (0,3), State at (3,3)]

# With obstacles
problem = ProblemOMSPP.Factory.with_obstacles()

# Custom: random grid with obstacles
problem = ProblemOMSPP.Factory.custom(rows=20, pct_obstacles=30, k=5)
Example 2: Decompose to SPP sub-problems 
problem = ProblemOMSPP.Factory.without_obstacles()
spps = problem.to_spps()
# spps[0] = ProblemSPP(grid, start, goal_a)
# spps[1] = ProblemSPP(grid, start, goal_b)
for spp in spps:
    print(spp.start, '->', spp.goal)
Example 3: Manual construction 
from f_ds.grids import GridMap as Grid
from f_search.ds.state import StateCell as State
from f_search.problems import ProblemOMSPP

grid = Grid(rows=10)
start = State(key=grid[0][0])
goals = [State(key=grid[0][9]), State(key=grid[9][9])]
problem = ProblemOMSPP(grid=grid, start=start, goals=goals)
