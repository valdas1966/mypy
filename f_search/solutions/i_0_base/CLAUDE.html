SolutionSearch

f_search/solutions/i_0_base/

A. Purpose

  1. Base solution class for all search algorithm results on grid problems.
  2. Stores algorithm name, problem reference, validity flag, and performance stats.
  3. Problem is stored as-is; pickle exclusion of heavy grid data is the problem's responsibility via its own __getstate__.

E. Usage Examples

Via Factory 
from f_search.solutions.i_0_base import SolutionSearch

# Create valid / invalid test solutions
valid = SolutionSearch.Factory.zero_valid()
invalid = SolutionSearch.Factory.zero_invalid()

assert valid.is_valid is True
assert invalid.is_valid is False
print(valid.name_algo)       # 'TestAlgo'
print(valid.stats.explored)  # stats metric
Design Note — Problem Storage
  1. SolutionSearch.__init__ stores the problem as-is (no to_light() call).
  2. The problem's __getstate__ handles excluding heavy grid data during pickle.
  3. After unpickling, call problem.load_grid(grids) to restore the grid reference.
