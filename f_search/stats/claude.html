Metric Meanings

elapsed

Total algorithm execution time. Measures overall speed/efficiency.

discovered

States added to open queue. Indicates search space coverage.

relaxed

States updated with better cost. Indicates path refinement.

explored

States fully expanded (moved to closed). Indicates actual search effort.

Metric Relationships

explored ≤ discovered relaxed ≤ discovered States can be: • Discovered once, never explored (discarded) • Discovered once, explored once (efficient) • Discovered once, relaxed N times, explored once (refinement)

Typical Values:

ScenarioPattern
Well-performing A*relaxed << discovered, explored ≈ discovered
Less efficient searchrelaxed → discovered, explored << discovered

Usage Examples

from f_search.stats import StatsSearch

# Create stats instance
stats = StatsSearch(
    elapsed=42,
    discovered=156,
    relaxed=23,
    explored=134
)

# Access metrics
print(f"Time: {stats.elapsed}ms")
print(f"Discovered: {stats.discovered}")
print(f"Relaxed: {stats.relaxed}")
print(f"Explored: {stats.explored}")

# Efficiency metrics
relax_ratio = stats.relaxed / stats.discovered
explore_ratio = stats.explored / stats.discovered

# Using Factory
test_stats = StatsSearch.Factory.linear()
