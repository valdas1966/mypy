9. Usage & Usefulness

  1. How to use: Subclass AlgoSPP, implement _discover() and _handle_successor(); _run() handles the loop.
  2. When useful: Any best-first SPP algorithm (A*, Dijkstra, weighted A*, etc.).
  3. Overengineered: No evidence of overengineering; the class is minimal at ~35 LOC.
  4. g_goal extraction: Cleanly separates path cost from path reconstruction; callers can get cost without building the full path.

  2. Low priority: Consider widening g_goal type to int | float | None if weighted grids are planned.
