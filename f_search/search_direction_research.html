Search Direction in Heuristic Search

Forward vs Backward on 2D Grids
f_search/search_direction_research.html — Literature Survey (March 2026)

Overview

This document surveys the academic literature on a fundamental question in heuristic search: given a start and goal on a 2D grid, is it better to search from start-to-goal or from goal-to-start?

Key Finding: There is no standalone paper that solves the problem of choosing forward vs. backward for unidirectional A* on 2D grids. This is an Open Research Gap.

Core Question

Forward Search: Start → → → → → Goal Backward Search: Start ← ← ← ← ← Goal Question: Which direction expands fewer nodes? Answer: Depends on local obstacle structure — but no published method exists.

Theoretical Background

Symmetry Theorem (Dechter & Pearl)

On undirected 2D grids with consistent heuristics (Manhattan/Octile), forward and backward A* are provably equivalent in terms of necessarily expanded nodes. The f-value distribution is symmetric, so the academic community has largely treated the direction question as moot for symmetric grids.

Branching Factor Principle

Classic Result (Poole & Mackworth): Search in the direction with the smaller branching factor. On uniform grids without obstacles, branching factors are symmetric (~4 or ~8 neighbors), so neither direction is favored.

Heuristic Quality Asymmetry

If the forward heuristic hF (state → goal) is more informed than the backward heuristic hB (state → start), or vice versa, the direction with the better heuristic will expand fewer nodes. On 2D grids with Manhattan or Octile distance, both directions use the same heuristic quality, so this factor is typically neutral.

Bidirectional Search Algorithms

Rather than choosing one direction, bidirectional search sidesteps the problem by searching in both directions simultaneously. The major algorithms developed by the Holte/Felner/Sturtevant research group:

MM — Meet in the Middle AAAI 2016

Bidirectional Search That Is Guaranteed to Meet in the Middle
Holte, Felner, Sharon, Sturtevant
AAAI 2016
The first bidirectional heuristic search algorithm guaranteed to meet in the middle — it never expands a node beyond the solution midpoint. Uses front-to-end heuristics.

NBS — Near-Optimal Bidirectional Search IJCAI 2017

Front-to-End Bidirectional Heuristic Search with Near-Optimal Node Expansions
Chen, Holte, Sturtevant, Felner
IJCAI 2017
Derives a lower bound VC on the minimum necessary expansions for any bidirectional search with consistent heuristics, and guarantees expanding at most 2×VC nodes. Proven to be near-optimal (no admissible front-to-end algorithm has a worst case better than 2×VC).

BAE* — Bidirectional A* SOCS 2025

Position Paper: On the Impact of Direction-Selection in BAE*
Shperberg, Siag, Sturtevant, Felner
SOCS 2025
Reviews direction-selection strategies in BAE* (Bidirectional A* with consistent heuristics), analyzing how the choice of which direction to expand next impacts performance. Presents a fix to prevent expanding nodes with f > C* across all direction-selection strategies.

SFBDS — Single-Frontier Bidirectional Search AAAI 2020

Single-Frontier Bidirectional Search
Moldenhauer, Felner
AAAI 2020
Uses a single open list containing pairs (s, g) representing subproblems, and recursively decides whether to expand from the start or goal side. Exploits irregularities in branching factor to reduce total work. Closest existing work to per-node direction selection.

Most Relevant Paper

Predicting the Effectiveness of Bidirectional Heuristic Search
Sturtevant, Shperberg, Felner, Chen
ICAPS 2020
Directly addresses when bidirectional search outperforms unidirectional A*. Identifies "critical states" in the state space as the primary factor — states that one direction must expand but the other need not. Proposes two simple, computationally inexpensive measurements:
  • M1: Performs a backward Dijkstra from the goal to a limited distance and counts states seen
  • M2: Samples random states and checks heuristic values
Why it matters: This paper provides a practical, computationally cheap way to predict whether searching in both directions will help — the closest published work to "which direction is better?"

Current State of the Art

For Undirected 2D Grids (Consistent Heuristics)

Forward and backward A* are provably equivalent in terms of necessarily expanded nodes. The choice does not matter theoretically. Bidirectional algorithms (NBS, MM) can outperform unidirectional A*, but only when there exist "critical states."

For Domains with Asymmetric Branching

Search in the direction with lower branching factor, or use bidirectional search. BAE* with appropriate direction-selection policies (per the 2025 SOCS paper) is the current best framework.

Practical Recommendation

On standard grid-based pathfinding, unidirectional A* with JPS (Jump Point Search) remains highly competitive. Bidirectional search helps most when heuristics are weak relative to true distance. When heuristics are strong (octile on grids), the savings from bidirectional search are modest.

Research Gap

Open Problem: The specific question "given a start and goal on a 2D grid with obstacles, should I search forward or backward?" has not been answered in a standalone paper.

The existing literature either:

  1. Assumes symmetric undirected graphs where the question is moot, or
  2. Studies bidirectional search, which sidesteps the question by searching in both directions simultaneously
Novel Contribution Opportunity: A paper that analyzes local obstacle structure around start/goal to predict which unidirectional direction will expand fewer nodes would be novel. The closest existing work is SFBDS, which makes per-node decisions about which direction to expand in a bidirectional framework.
