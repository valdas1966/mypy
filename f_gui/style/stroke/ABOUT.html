What is a Stroke?

The look of a line — color, width, style — with no geometry, reused everywhere a stroke appears.

value object pure appearance + LineStyle enum

The Idea

A pen-tip setting you configure once, then reuse.

🖊️

A Stroke is a pen-tip setting: choose the ink color, the nib width, and the dash pattern once — then use that same pen to draw many lines or trace many border edges. The pen knows nothing about where you draw; it only knows how the ink looks.

🚫 Without Stroke

  • The (color, width, style) triple is repeated on Line and again on every Border edge.
  • Four border sides + lines = the same three fields copied many times.
  • Values drift: a width tweak on one edge silently diverges from the rest.
  • Duplication with no single source of truth.

✅ With Stroke

  • One Stroke value object holds the triple exactly once.
  • Both Line and each Border edge reuse the same Stroke type.
  • Change appearance in one place — every consumer agrees.
  • A small immutable bundle, shared by reference.
  1. Pure appearance, no geometry. A Stroke never stores endpoints, sides, or positions — only how the ink looks.
  2. color=None means renderer default. When color is None, the renderer supplies the fallback — the Stroke does not.
  3. width is in pixels (a float, default 1).
  4. style is a LineStyle — SOLID / DASHED / DOTTED.
  5. LineStyle values are the exact CSS keywords ('solid' / 'dashed' / 'dotted'), so the renderer maps them with zero translation.
  6. RGB is imported only under TYPE_CHECKING, so a plain Stroke never pulls in matplotlib.

This Module's Contribution

The single reuse point for line / edge appearance — plus the LineStyle enum that both share.

f_gui/style/stroke/main.py 

class LineStyle(Enum):
    # Values are the exact CSS keywords (border-style / SVG dash
    # family), so the renderer maps them with zero translation.
    SOLID  = 'solid'
    DASHED = 'dashed'
    DOTTED = 'dotted'


class Stroke:
    # Pure appearance, no geometry.

    def __init__(self,
                 color: RGB | None = None,
                 width: float = 1,
                 style: LineStyle = LineStyle.SOLID) -> None:
        self._color = color
        self._width = width
        self._style = style

    def __str__(self) -> str:
        color = self._color if self._color is not None else 'default'
        return f'({self._width}px {self._style.value} {color})'

key idea Defining the triple here — once — is what lets Line and every Border edge agree on appearance without copying fields.

Fun Facts

Accurate corners of the stroke world.

🖌️

Stroke vs. fill

In vector graphics, the stroke is the outline of a shape — the counterpart of the fill, which is its interior.

┄

stroke-dasharray

The SVG attribute that defines a dash pattern as a sequence of on/off lengths along the line.

🔤

No mapping needed

CSS border-style keywords are literally solid/dashed/dotted — the enum stores them verbatim.

🪶

The flyweight idea

A value object is a small immutable bundle of data shared by reference — one Stroke, many users.

📏

Sub-pixel widths

Width is a float, so sub-pixel strokes (e.g. 0.5) are expressible.
