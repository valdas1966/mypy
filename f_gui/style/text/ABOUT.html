What is a TextStyle?

How text looks — font, size, bold, color — kept separate from the words themselves.

value object pure presentation opt-in & reusable

The Idea

Same words, many looks — one look, many runs of text.

✍️

A TextStyle is a character / font style in a word processor. The same words can wear many looks (a heading, a caption, a code snippet), and one look can dress many runs of text across a document. The style is a thing you define once and reuse — never something baked into the letters you typed.

🚫 Look baked into the content

Font, size, and color welded onto the text — or onto the Label itself.

  • Change the look ⇒ touch every Label.
  • Content and presentation tangled together.
  • No way to share one look across many Labels.

✅ A reusable TextStyle value object

A standalone TextStyle attached to a Label the same opt-in way as background / border.

  • The str (content) stays on the Label.
  • The look is separate, named, and reused.
  • Sits beside Stroke / Border in f_gui.style.

The rules it lives by

  1. Pure presentation, no content. A TextStyle never holds the text string — that belongs to the Label.
  2. Sensible defaults: monospace, 12px, not bold.
  3. Default color is black — readable on a light background.
  4. Explicit color=None opts OUT of a color and inherits the surrounding page color instead.
  5. A Label with style=None renders the same as TextStyle() — the black baseline.
  6. RGB is imported lazily inside __init__, so f_color stays off TextStyle's module-load path.

This Module's Contribution

The three-state color design — and the sentinel that makes it possible.

main.py sentinel A module-level sentinel distinguishes "omitted → black" from "explicit None → inherit" — three states, not two.

_DEFAULT_BLACK: Any = object()  # sentinel: color not passed


class TextStyle:

    def __init__(self,
                 font: str = 'monospace',
                 size: float = 12,
                 bold: bool = False,
                 color: RGB | None = _DEFAULT_BLACK) -> None:
        if color is _DEFAULT_BLACK:
            from f_color.rgb import RGB  # lazy import
            color = RGB('black')
        self._font  = font
        self._size  = size
        self._bold  = bold
        self._color = color

    def __str__(self) -> str:
        weight = 'bold' if self._bold else 'normal'
        color  = self._color if self._color is not None else 'inherit'
        return f'({self._font} {self._size}px {weight} {color})'

Using it

from f_gui.elements.i_1_label import Label
from f_gui.style.text import TextStyle
from f_color.rgb import RGB

Label(text='Title', style=TextStyle.Factory.title())
Label(text='note', style=TextStyle(size=14, color=RGB(name='RED')))
Label(text='plain')                              # style=None -> baseline black
Label(text='on dark', style=TextStyle(color=None))  # inherit page color

Fun Facts

Why presentation, sentinels, and monospace are the way they are.

🎭

Content vs. presentation

Separating the two is the founding idea of CSS (1996): one document, many looks. TextStyle is that same split, in miniature.

🔀

Three states need a sentinel

Omitted → black, None → inherit, an RGB → that color. Python can't tell "not passed" from "passed None" — so a sentinel object is required.

⌨️

Monospace heritage

monospace — every glyph the same width — is the typewriter legacy, and TextStyle's baseline default.

🏋️

font-weight: words or numbers

CSS font-weight accepts both keywords (bold) and numbers (100–900).

🪶

Lazy import = light module

Importing RGB inside __init__ keeps the heavy f_color / matplotlib dependency off the module-load path.
