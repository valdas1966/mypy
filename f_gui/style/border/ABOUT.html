What is a Border?

Four edge Strokes around a box — each side independent, each optional.

four edges composes Stroke fully opt-in

The Idea

A frame you assemble rail by rail.

🖼️

A Border is a picture frame built from four separate rails — top, left, bottom, right. You can leave a side off entirely, or make each rail look different: a dashed crown across the top, a heavy line under the bottom, bare wood on the sides. The frame is never one rigid piece; it is four pieces you choose independently.

🚫 One monolithic border

A single setting forced onto all four sides:

  • Want a dashed top but a solid bottom? Impossible.
  • Want no sides at all? Impossible.
  • One knob controls everything — uniformity is mandatory.

✅ Four independent Strokes

Each side is its own Stroke — or nothing:

  • Dash the top, make the bottom solid.
  • Omit the left and right — they emit nothing.
  • Each edge reuses the same appearance type as a Line.
  1. Each side — top, left, bottom, right — is a Stroke or None.
  2. None means no edge on that side — nothing is drawn there.
  3. A Border is opt-in Element state (Element.border) — not a scene-graph node.
  4. It renders to one CSS border-{side} declaration per set side.
  5. Stroke is imported under TYPE_CHECKING in main.py — eager only in the factory.

A border edge and a Line speak the same appearance vocabulary — both are made of Stroke. A Border is just four Strokes grouped and attached to an Element as its border property. The (color, width, pattern) triple is defined exactly once.

This Module's Contribution

The grouping of four edges — and nothing more.

main.py class Border 
class Border:
    def __init__(self,
                 top:    Stroke | None = None,
                 left:   Stroke | None = None,
                 bottom: Stroke | None = None,
                 right:  Stroke | None = None) -> None:
        self._top    = top
        self._left   = left
        self._bottom = bottom
        self._right  = right

    @property
    def top(self) -> Stroke | None:
        # None = no top border
        return self._top

    def __str__(self) -> str:
        sides = (('T', self._top),  ('L', self._left),
                 ('B', self._bottom), ('R', self._right))
        parts = [f'{tag}={s}' for tag, s in sides if s is not None]
        return f'Border[{", ".join(parts)}]' if parts else 'Border[]'

Callout — Border groups four Strokes behind four matching properties (top/left/bottom/right), each Stroke | None. Fully opt-in: __str__ prints only the sides that are set.

Fun Facts

Five things worth knowing.

🧭

CSS has four side properties

The web mirrors this model exactly: border-top, border-right, border-bottom, border-left — four independent declarations.

📦

The box model order

CSS wraps content in padding, then border, then margin — the border sits between the inner padding and the outer margin.

🫥

Borders are fully opt-in

When border is None, nothing is emitted — the old per-type default borders were removed.

🧩

Composition over a monolith

A Border is literally four Strokes, reusing the one appearance type shared with Line.

🖨️

__str__ prints only set sides

A top-only border reads Border[T=(2px solid default)] — unset sides never appear.
