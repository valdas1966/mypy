What is a Label?

A leaf that carries text — with content and styling kept deliberately separate.

leaf element · carries text

The Idea

A Label is the simplest possible element: it just holds some words.

🏷️

Think of a printed sticky note. The words on it are one thing — "Hello", "Save", "Total". The font they're printed in is something else entirely. Swap the marker for a fountain pen and the message doesn't change; rewrite the message and the ink color stays. A Label keeps those two ideas apart on purpose: the text is what it says, the style is how it looks.

🚫 One tangled blob

  • The string and its font, size, and color all welded into a single object.
  • Want the same words in a new look? You rebuild everything.
  • Presentation logic leaks into the content.

✅ Content ⟂ presentation

  • text is a plain str — just the words.
  • style is an optional TextStyle — the look, opt-in.
  • Exactly how background / border are separate on every Element.

The contract

  1. A Label is a leaf — it holds no children, it is the end of a branch.
  2. text is a plain str — nothing fancier than the words themselves.
  3. On render the text is HTML-escaped and centered in the label's box.
  4. style=None falls back to the renderer baseline look — monospace, 12px.
  5. A Label carries exactly one TextStyle — no style stacking.

How It Connects

Label is the leaf that sits one step below the abstract Element base.

Element # i_0_element — abstract base: bounds + name + parent └── Label # i_1_label — leaf, carries text ◀ you are here
🧩

The TextStyle itself lives in f_gui.style.text, not here. That split is intentional: the content module (Label) and the presentation module (TextStyle) are kept apart, so a look can be defined once and reused across many labels.

This Module's Contribution

A leaf Element with a text string and an optional, separate style.

main.py  The real constructor — note text and style live side by side as independent parameters:

class Label(Element):

    def __init__(self,
                 bounds: Bounds[float] = None,
                 text: str = '',
                 name: str = 'Label',
                 background: RGB | None = None,
                 border: Border | None = None,
                 style: TextStyle | None = None) -> None:
        Element.__init__(self, bounds=bounds, name=name,
                         background=background, border=border)
        self._text = text     # the words — a plain str
        self._style = style   # the look — optional TextStyle

properties  Read-only access to each, kept separate:

    @property
    def text(self) -> str:
        return self._text

    @property
    def style(self) -> TextStyle | None:   # None = renderer default look
        return self._style

Fun Facts

Five ideas baked into this tiny leaf.

🎨

The founding idea of CSS

Separating content from presentation is exactly why CSS was born in 1996 — so one document could take on many looks.

🛡️

Escaping defuses markup

HTML-escaping turns text like <b> into literal characters instead of tags — the root of defending against XSS.

⌨️

Monospace by default

The baseline look gives every glyph the same width — a straight inheritance from the typewriter era.

🎯

One style, no stacking

A Label carries exactly one TextStyle. The deliberate "no stacking" choice keeps the model simple to reason about.

🍃

A leaf has no children

With nothing below it, a Label is the natural terminator of the element tree — a branch that simply ends.
