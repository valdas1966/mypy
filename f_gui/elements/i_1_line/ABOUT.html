What is a Line?

A directed segment from p1 to p2 — the one element that renders as SVG, not a box.

leaf Element two Points Stroke appearance optional arrowhead

The Idea

Every other element is a rectangle. A Line is a stroke.

✏️

A Line is a pencil stroke drawn between two pins. You pin one end at p1, the other at p2, and drag the pencil straight across. You simply cannot draw a clean diagonal with a rectangle — boxes have only horizontal and vertical edges, so a slanted stroke needs a different tool.

❌ Faking a diagonal with boxes

Rotate and clip a thin <div> to look slanted.

  • Fragile — rotation math drifts at every zoom level
  • No real dash pattern (a clipped box has no dashes)
  • No arrowhead — you'd glue on a second rotated box
  • Endpoints become approximate, not exact p1/p2

✅ An inline <svg> overlay

The only pure-HTML way to draw a true diagonal.

  • Exact endpoints as percentages — maps 1:1 onto the parent
  • Real dashes via stroke-dasharray
  • A real arrowhead via an SVG <marker>
  • Crisp at any zoom — it's a vector, not pixels

The rules that define a Line

  1. A Line is defined by two Points, p1 -> p2, in the normalized 0–100 space (relative to the parent).
  2. Appearance is a Stroke = (color, width, style) — the same value object a Border edge uses, so the appearance triple lives in one place.
  3. An optional arrowhead sits at p2, the end point (so direction is well-defined: p1 → p2).
  4. The Element bounds are the bounding box of the two endpoints — the min/max of each axis.
  5. The inherited border is ignored — a Line renders as SVG, not a bordered div.

How It Connects

A leaf element that swaps the box for a stroke.

Element (abstract — bounds + name + parent) └── Line i_1_line (leaf; two Points + Stroke + arrow)
🔗

Stroke and LineStyle (SOLID / DASHED / DOTTED) come from f_gui.style.stroke — shared with Border. A Line and a Border edge describe their appearance with the exact same triple; only the geometry differs.

This Module's Contribution

Geometry is Line's own; appearance is borrowed.

def __init__(self,
             p1: Point,
             p2: Point,
             stroke: Stroke | None = None,
             arrow: bool = False,
             name: str = 'Line') -> None:
    Element.__init__(self, bounds=Line._bbox(p1=p1, p2=p2), name=name)
    self._p1 = p1
    self._p2 = p2
    self._stroke = stroke or Stroke()
    self._arrow = arrow

@staticmethod
def _bbox(p1: Point, p2: Point) -> Bounds[float]:
    # bounds = the bounding box of the two endpoints
    return Bounds(top=min(p1.y, p2.y),
                  left=min(p1.x, p2.x),
                  bottom=max(p1.y, p2.y),
                  right=max(p1.x, p2.x))

Callout: the geometry (p1, p2, the bounding box, the arrowhead) is Line's own; the appearance (Stroke) is shared with Border. stroke or Stroke() means a missing stroke falls back to the default.

Fun Facts

The vector graphics behind the stroke.

🌐

SVG is a W3C standard

Scalable Vector Graphics is an XML language for 2-D vectors that scale without blur.

🔍

Vector vs raster

A Line stored as two points re-renders crisply at any zoom — unlike pixels, which blur when scaled.

🚫

Boxes can't go diagonal

HTML/CSS boxes can't draw true diagonals — SVG (or canvas) is required.

┄

stroke-dasharray

This SVG attribute defines the dash pattern — the lengths of dashes and gaps along the stroke.

➤

Direction is built in

A directed segment carries orientation (p1→p2), which is why the arrowhead has a well-defined end.
