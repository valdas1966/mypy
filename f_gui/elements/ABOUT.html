The 0–100 coordinate model

Every Bounds is relative to its parent, in a normalized 0–100 box — so it maps 1:1 onto CSS percentages.

🗺️

Think percentages, not pixels. A child at left=10, right=30 always sits in the left tenth-to-third of whatever its parent is — resize the parent and the child scales with it. That is why the whole renderer is ~30 lines: 0–100 ⇒ CSS %, no layout math.

  1. Bounds = (top, left, bottom, right) — the four edges, each 0–100, top-left origin.
  2. Relative to parent — a label's bounds are read inside its container, not the window.
  3. anchor(side) returns the mid-point of an edge (TOP / RIGHT / BOTTOM / LEFT) — where a Connector attaches.
  4. Window is implicitly full — (0,0,100,100); it never takes a bounds argument.

from f_gui.elements.i_2_window    import Window
from f_gui.elements.i_1_container import Container
from f_gui.elements.i_1_label     import Label
from f_gui.elements.i_1_connector import Connector
from f_ds.geometry.bounds         import Bounds

win   = Window.Factory.default()
a     = Container(bounds=Bounds(top=20, left=10, bottom=40, right=30))
b     = Container(bounds=Bounds(top=60, left=70, bottom=80, right=90))
link  = Connector(src=a, dst=b)            # auto sides, arrow at dst

win.add_child(child=a); win.add_child(child=b); win.add_child(child=link)
win.to_html(path='/tmp/demo.html')      # the whole tree follows

  


  
  

    
Why it's built this way

    

      
🌳
Retained mode

        
The scene is kept as a tree of objects you mutate — not redrawn from scratch each frame
          (immediate mode). The same model the DOM and SVG use.

      
🧩
Composite pattern

        
Container/leaf is the Gang-of-Four Composite: treat one element and a whole subtree the
          same way through one Element interface.

      
🧲
Derived, never stale

        
A Connector's path and bounds are computed
          on every read from the live element bounds — so they can't drift out of sync.

      
📐
Normalized space

        
0–100 relative coordinates echo SVG's viewBox and normalized
          device coordinates: layout that's resolution-independent by construction.

      
✂️
Data ⟂ rendering

        
The elements store structure only. Hit-testing, focus, and events are out of scope; one
          stateless emitter turns the tree into HTML.

      
📏
Inheritance as folders

        
The i_X_ folder prefix encodes depth: i_0
          base, i_1 the family, i_2 Window.
