What is an Element?

The abstract node every GUI thing inherits from — it owns identity, position, and shape, but can never be drawn itself.

The Idea

One blueprint, many concrete shapes cut from it.

🧬

Element is a cookie-cutter, not a cookie. It defines the shape that every concrete GUI thing shares — a bounds rectangle, a name, a parent pointer, and four anchor points — but you can't bake the cutter. Calling Element() directly raises TypeError. You always instantiate a concrete subclass: a Container, a Label, a Window, and so on.

📐

Bounds

A rectangle in a normalized 0–100 space, relative to the parent. Defaults to the full region (0, 0, 100, 100) via Bounds.Factory.full().

🏷️

Identity

Composed from the HasName and HasParent mixins → a name, a parent pointer, and path_from_root() up the tree.

🔗

Anchor

anchor(side) returns the mid-point of an edge (TOP / RIGHT / BOTTOM / LEFT) — the spot where a Connector latches on.

The rules that make it tick

  1. Abstractness is enforced by a __new__ guard, not abc.ABC — if cls is Element: raise TypeError(...).
  2. Bounds default to full (0, 0, 100, 100) when none is passed.
  3. background and border are opt-in, both default to None (transparent / no border).
  4. anchor(side) delegates to self.bounds.anchor(side) and reads the bounds property — so subclasses with dynamic bounds stay correct.
  5. It is a pure data model: rendering lives in RenderHtml, never here.

How It Connects

Element sits between the f_core mixins and every concrete widget.

f_core mixins HasName , HasParent └── Element (abstract — i_0_element) ├── Container i_1_container │ └── Window i_2_window (only depth-2 class) ├── Label i_1_label ├── Line i_1_line └── Connector i_1_connector

Every concrete element is an Element; Window is the only depth-2 class (a Container that is itself an Element).

This Module's Contribution

The abstractness guard, the constructor, and the anchor — straight from main.py.

__new__ guard no metaclass no fake abstract method 

def __new__(cls, *args, **kwargs) -> 'Element':
    if cls is Element:
        raise TypeError(
            'Element is abstract; instantiate a concrete subclass '
            '(Container, Label, Window).')
    return super().__new__(cls)

This single check blocks Element() while allowing every subclass (where cls is not Element) — no metaclass, no fake @abstractmethod hook.

constructor — subclasses only

def __init__(self,
             bounds: Bounds[float] = None,
             name: str = 'Element',
             background: RGB | None = None,
             border: Border | None = None) -> None:
    HasName.__init__(self, name=name)
    HasParent.__init__(self)
    self._bounds = bounds or Bounds.Factory.full()
    self._background = background
    self._border = border

anchor — reads the property

def anchor(self, side: Side) -> Point:
    # mid-point of the given edge; where a Connector attaches
    return self.bounds.anchor(side=side)

Fun Facts

Context that makes the design choices click.

🌳

Retained, not immediate

f_gui stores a tree of objects you can inspect and mutate (retained mode) — unlike immediate-mode GUIs that redraw from raw draw-calls each frame.

🕸️

Scene graphs are everywhere

The same parent/child node tree shows up in SVG / the DOM, game engines, and vector editors like Illustrator or Figma.

⚠️

The ABC gotcha

Subclassing abc.ABC without an @abstractmethod does not block instantiation — exactly why Element uses a __new__ guard instead.

🧬

Mixins + MRO

Element composes HasName and HasParent as capability mixins; Python's MRO linearizes them so each __init__ runs once.

🗺️

Normalized 0–100 space

Resolution-independent coordinates echo SVG's viewBox and normalized device coordinates — layout stays correct at any pixel size.
