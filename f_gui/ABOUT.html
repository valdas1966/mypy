🧬 Class Hierarchy

i_0 Element

Bounds + name + parent pointer. Every node IS-A Element.

i_1 Container

Holds children. Owns parent/children invariants.

i_1 Label

Leaf with text. No children.

i_2 Window

Root container with implicit full bounds.

📐 Bounds Coordinate System

All positions are relative to the parent in a normalized 0-100 space — resolution-independent.

Example Layout

Window (0,0,100,100)
Container (30,50,50,70)
Label[Hello]

Tuple format

bounds.to_tuple() # (top, left, bottom, right)

Window    → (0,  0,  100, 100)
Container → (30, 50, 50,  70)  # inside Window
Label     → (20, 10, 40,  30)  # inside Container
Child bounds are local. The label at (20,10,40,30) means "20-40% from the Container's top, 10-30% from its left" — not window coordinates.

🌳 Tree-Mutation Semantics (★ new)

OperationResult
container.add_child(child)
— fresh child		 Sets child.parent = container via _set_parent(); appends to children.
container.add_child(child)
— child owned by another parent		 Detaches from old parent first (old.remove_child(child)), then attaches here. No orphaned entries.
container.add_child(child)
— child already owned by this container		 No-op. Prevents duplicate entries in children.
container.remove_child(child) Drops from children and clears child.parent via _set_parent(None). Raises ValueError if not present.

Reparent Flow

old.children = [child] → new.add_child(child) → old.remove_child(child) → child._set_parent(new) → new._children.append(child)
Invariant preserved: every child appears in exactly one children list, and child.parent.children always contains child.

💡 Usage Examples

1 · Create a simple element

from f_gui.elements.i_0_element import Element

e = Element.Factory.full()           # bounds (0, 0, 100, 100)
print(e)                             # Element(0, 0, 100, 100)
print(e.bounds.to_tuple())           # (0, 0, 100, 100)

2 · Build a Window → Container → Label tree

from f_gui.elements.i_2_window    import Window
from f_gui.elements.i_1_container import Container
from f_gui.elements.i_1_label     import Label
from f_ds.geometry.bounds         import Bounds

win   = Window.Factory.default()
panel = Container(bounds=Bounds(top=30, left=50,
                                bottom=50, right=70))
hello = Label(bounds=Bounds(top=20, left=10,
                            bottom=40, right=30),
              text='Hello')

win.add_child(child=panel)
panel.add_child(child=hello)

assert hello.parent is panel
assert panel.parent is win
assert win.parent   is None

3 · Reparent a child (★ new behavior)

old = Container.Factory.full()
new = Container.Factory.half()
lbl = Label.Factory.hello()

old.add_child(child=lbl)
assert old.children == [lbl]

# Moving the label to `new` auto-detaches it from `old`.
new.add_child(child=lbl)
assert lbl.parent   is new
assert old.children == []
assert new.children == [lbl]

4 · Remove a child (★ new)

panel.remove_child(child=hello)
assert hello.parent is None
assert panel.children == []

5 · Walk the tree from a leaf back to the root

# path_from_root() is inherited from HasParent
for node in hello.path_from_root():
    print(node.name)
# Window → Container → Label

6 · Use ready-made Factory presets

Element.Factory.full()       # (0, 0, 100, 100)
Element.Factory.half()       # (25, 25, 75, 75) centered
Container.Factory.full()     # full-parent container
Container.Factory.half()     # ★ now reuses Bounds.Factory.half()
Label.Factory.hello()        # Label[Hello]
Label.Factory.title()        # top 10% title bar
Window.Factory.default()     # root window

🛠️ Implementation Sketch

HasParent — protected setter

def _set_parent(self, parent: Self | None) -> None:
    """Protected helper for friend-classes."""
    self._parent = parent

HasChildren — remove counterpart

def remove_child(self, child: Self) -> None:
    # Raises ValueError if not present.
    self._children.remove(child)

Container — orchestrates invariants

def add_child(self, child: Element) -> None:
    old_parent = child.parent
    if old_parent is not None:
        if old_parent is self:
            return                             # no-op
        old_parent.remove_child(child=child)    # detach
    child._set_parent(parent=self)              # encapsulated
    HasChildren.add_child(self, child=child)

def remove_child(self, child: Element) -> None:
    HasChildren.remove_child(self, child=child)
    child._set_parent(parent=None)

🧠 Design Review — v2

✅ Strengths

  • Framework-idiomatic. i_X_ folders, Factory wiring, mixin composition, =-docstrings — all per CLAUDE.md.
  • Encapsulation is clean. Parent mutation flows through _set_parent(); no consumer pokes private state.
  • Reparent-safe. Moving a child between containers never leaves dangling references; duplicate adds are a no-op.
  • Resolution-independent bounds (0-100 relative space).
  • Lazy aggregator __init__.py. No cascade imports — importing Label doesn't load the rest.
  • Tests are direct. No fixture ceremony for one-line passthroughs; each test is self-contained.
  • Complete CLAUDE.md coverage at every level of the package.

⚠️ Remaining / Future work

  • No rendering layer. This is a scene graph, not a renderer. Document this clearly if third parties consume it.
  • No visibility / z-order / enabled flags — expected next mixins (HasVisible, HasZOrder).
  • No hit-testing / coordinate-resolution helpers. A bounds_in_root() that composes the parent chain would be useful.
  • Window.__init__ drops bounds. Still implicit — consider an explicit bounds=Bounds.Factory.full() default.
  • No event / observer hooks (on_child_added, on_removed) — needed once a renderer plugs in.
  • Integrity checks (cycles, orphans, etc.) live out-of-band. Kept out of the hot path by design — validated by a separate debugging tool, not inline guards.
Bottom line. v2 closes the encapsulation and lifecycle gaps called out in v1. The package is a solid, minimal foundation for a retained-mode scene graph — next moves are visibility, hit-testing, and a rendering adapter.
