What is a LaTeX Compiler?

Source code for documents. You write meaning; the compiler designs the page.
/mnt/f/mypy/f_tex/ — ABOUT.html

The Idea

🖨️

Think of it like a C compiler — but for pages.

You write .tex source describing what the document means (a section, a theorem, a figure). The compiler decides how it looks — fonts, kerning, hyphenation, page breaks, floats, references. Same source, same PDF, forever. It doesn't matter what laptop you compile it on.

WYSIWYGWord / Docs

  • You arrange pixels by hand.
  • Break if the font changes.
  • Hard to diff, hard to version.
  • "Looks fine" is the quality bar.

WYSIWYMLaTeX

  • You declare structure, not position.
  • Re-flows automatically on any medium.
  • Plain text — git-diff friendly.
  • The engine is responsible for typography.

Rules of the game

  1. Semantic markup. You mark \section, not "bold 18pt, 12pt space above."
  2. Reproducible. Same source + same engine + same packages → byte-identical PDF.
  3. Templates are code. Preambles, macros and classes are the stylesheet — and they compose.
  4. Floats and references are a mini-language. You label; the engine places and numbers.
  5. The compiler is slow because it thinks. Line-breaking is a constrained optimization over the whole paragraph (Knuth–Plass).

What compile() actually does

  1. Opens a tempfile.TemporaryDirectory.
  2. Writes the source string to main.tex.
  3. Spawns tectonic --chatter=minimal main.tex.
  4. Reads main.pdf back as bytes.
  5. Lets the temp dir self-destruct on exit — no junk left behind.

The LaTeX Engine Family

TeX 1978

The original. Knuth. Still runs. Defines the algorithms.

pdfTeX 1997

TeX that writes PDF directly (no DVI step). The default on most distros.

XeTeX 2004

Unicode + system fonts. If you want Arabic, Chinese, or OpenType features, use this.

LuaTeX 2007

Embeds Lua. Programmable typesetting. Successor heir-apparent.

ConTeXt 1996

Alternative macro layer. Different philosophy from LaTeX — full document class replacement.

tectonic 2016+

Rust rewrite of XeTeX. Self-contained. Auto-fetches packages. The default for f_tex.

Fun Facts & History

📐

Born from Knuth's frustration

Donald Knuth began TeX in 1977 because the galley proofs of his book The Art of Computer Programming looked bad after the publisher switched to photo-typesetting. He took a "one-year" detour to build better typesetting. It took ten.

💰

The $2.56 bug bounty

Knuth pays $2.56 for every TeX bug reported — "one hexadecimal dollar" (256 cents). The amount doubled yearly early on, reaching $327.68. Most recipients frame the check instead of cashing it.

🥧

TeX's version number converges to π

TeX versions add a decimal digit: 3.14 → 3.141 → 3.1415 → 3.14159265. Knuth declared that upon his death the version freezes at π and any remaining bugs become features. METAFONT's version converges to e similarly.

🔠

It's "LAH-tek," not "LAY-tex"

The X is the Greek letter χ (chi) — so the final sound is "k," not "x." Knuth and Lamport insist on either "LAH-tek" or "LAY-tek." The rubber material has nothing to do with it.

🧠

TeX is Turing-complete

People have implemented Conway's Game of Life, a Lisp interpreter, even a TCP client — entirely in TeX macros. Your preamble is, in principle, a programming environment. In practice, please don't.

📏

Line-breaking is an optimization problem

TeX's paragraph builder (the Knuth–Plass algorithm) solves global line-breaking via dynamic programming — minimising total "badness." That's why the same paragraph can look markedly better in TeX than in editors that break greedily, line-by-line.
