f_tex — Tex

/mnt/f/mypy/f_tex/

A. Purpose

  1. Thin facade around a local LaTeX engine (tectonic by default) that compiles .tex source or files into PDF bytes.
  2. Ships one opinionated template — template_session_summary — matching the Drive For_Session_Summary.md conventions (orange accent, fancyhdr header, \unchecked command).
  3. Designed to compose with f_google.services.drive: read .tex from Drive, compile locally, upload the PDF back.

B. Public API

class Tex

__init__ constructor

__init__(self, engine: str = 'tectonic') -> None
  1. Bind the compiler to a LaTeX engine binary resolvable on PATH.
  2. Default tectonic; pass another engine name to override.

engine property

engine: str
  1. Returns the engine binary name (e.g. 'tectonic').

compile method

compile(self, src: str) -> bytes
  1. Compiles a .tex source string and returns the PDF as bytes.
  2. Runs the engine inside a tempfile.TemporaryDirectory.
  3. No artifacts left on disk after return.
  4. Raises RuntimeError with engine stderr on compile failure.
  5. Raises RuntimeError if the engine binary is missing from PATH.

compile_file method

compile_file(self,
             path_src: str,
             path_dest: str | None = None) -> bytes
  1. Compiles a .tex file on disk; returns PDF bytes.
  2. \input{..} and relative image paths resolve from path_src's directory (multi-file projects work).
  3. Writes the PDF to path_dest if given; pure read otherwise.
  4. Raises FileNotFoundError if path_src does not exist.
  5. Raises RuntimeError on compile failure.

__repr__ dunder

__repr__(self) -> str
  1. Returns Tex(engine='tectonic')-style repr.
class Templates static
  1. Static .tex source builders — no state, no instance.
  2. Separate from Tex to keep the compiler single-responsibility.
  3. Lives in _templates.py; re-exported by __init__.py.

session_summary static

session_summary(project_name: str,
                date: str,
                project_path: str,
                body: str) -> str
  1. Returns a complete .tex source for a session summary.
  2. body is raw LaTeX injected after the title block.
  3. Template matches Drive For_Session_Summary.md: orange accent (#E65100), fancyhdr header, \unchecked command.
  4. Placeholders __PROJECT_NAME__, __DATE__, __PROJECT_PATH__, __BODY__ are substituted verbatim.

E. Usage examples

Example 1 — Source string → PDF bytes
  1. One-line compile, bytes in memory, no disk artifacts.
from f_tex import Tex

tex = Tex.Factory.a()
pdf = tex.compile(src=r'''
\documentclass{article}
\begin{document} Hello. \end{document}
''')
assert pdf[:4] == b'%PDF'
Example 2 — Compile a file and write the PDF
  1. Same as running tectonic locally, minus the cwd dance.
from f_tex import Tex

tex = Tex.Factory.a()
tex.compile_file(
    path_src='/tmp/paper.tex',
    path_dest='/tmp/paper.pdf',
)
Example 3 — Round-trip through Google Drive
  1. Read source from Drive, compile locally, upload the PDF.
  2. This is the composition pattern f_tex exists to enable.
from f_google.services.drive import Drive
from f_tex import Tex

drive = Drive.Factory.valdas()
tex   = Tex.Factory.a()

src = drive.read(path='2026/04/16/drive_session.tex').text
pdf = tex.compile(src=src)

open('/tmp/drive_session.pdf', 'wb').write(pdf)
drive.upload(
    path_src='/tmp/drive_session.pdf',
    path_dest='2026/04/16/drive_session.pdf',
)
Example 4 — Session-summary template
  1. Generates the boilerplate preamble and title block.
  2. The caller supplies only the body — sections and next-steps.
from f_tex import Tex, Templates

body = (
    r'\section{Purpose} Scaffolding session.'
    '\n'
    r'\section{Next Steps} \unchecked Ship it.'
)
src = Templates.session_summary(
    project_name='Drive',
    date='2026-04-16',
    project_path='/mnt/f/mypy',
    body=body,
)
pdf = Tex.Factory.a().compile(src=src)
