1. What is HTTP?

HTTP (HyperText Transfer Protocol) is the application-layer protocol that powers the World Wide Web. Every time you open a webpage, call a REST API, or stream a video, HTTP is the language your client speaks with the server.

📡

Request / Response

Client sends a request, server returns a response. Stateless — each exchange is independent.

📜

Text-Based

HTTP messages are human-readable text (headers + body). Even binary payloads ride on top of text-framed envelopes.

🔒

HTTPS

HTTP + TLS encryption. All modern traffic should be HTTPS — the "S" stands for Secure.

🌍

Universal

Runs on TCP port 80 (or 443 for HTTPS). Supported by every browser, every server, every language.

2. Brief History

1989 Tim Berners-Lee proposes the World Wide Web at CERN.
1991 HTTP/0.9 — one-line protocol. GET only. No headers, no status codes.
1996 HTTP/1.0 (RFC 1945) — headers, status codes, POST, Content-Type.
1997 HTTP/1.1 (RFC 2068) — persistent connections, chunked transfer, Host header.
2015 HTTP/2 (RFC 7540) — binary framing, multiplexing, header compression, server push.
2022 HTTP/3 (RFC 9114) — runs over QUIC (UDP), 0-RTT handshake, eliminates head-of-line blocking.

3. Request & Response Cycle

The client opens a TCP connection, sends a request line (method + path + version), followed by headers and an optional body. The server processes it and returns a status line, headers, and body.

4. HTTP Methods

GETRetrieve a resource. Safe, idempotent. Most common method.
POSTCreate a resource or submit data. Not idempotent.
PUTReplace an entire resource. Idempotent.
PATCHPartially update a resource.
DELETERemove a resource. Idempotent.
HEADLike GET but response has no body. Useful for checking existence.
OPTIONSDescribe communication options. Used in CORS preflight.

5. Status Codes

Every HTTP response carries a 3-digit status code grouped into five families:

RangeFamilyMeaningExamples
1xx Informational Request received, continuing 100 Continue, 101 Switching Protocols
2xx Success Request succeeded 200 OK, 201 Created, 204 No Content
3xx Redirection Further action needed 301 Moved Permanently, 304 Not Modified
4xx Client Error Bad request from client 400 Bad Request, 401, 403, 404, 429
5xx Server Error Server failed to fulfill 500 Internal, 502 Bad Gateway, 503 Unavailable

Famous Status Codes

200 OK

The golden standard. Everything worked.

301 Moved

Resource permanently relocated. Browsers follow automatically.

404 Not Found

The most famous error on the internet. Resource doesn't exist.

418 Teapot

RFC 2324 joke: "I'm a teapot". A real code since 1998.

429 Too Many

Rate limit hit. Slow down your requests.

503 Unavailable

Server overloaded or in maintenance. Try again later.

6. Headers

Headers carry metadata about the request or response. Key-value pairs separated by :.

8. f_http Modules

The f_http package wraps Python's requests library into three structured classes, turning raw HTTP into typed, validatable objects.

Client

Static HTTP GET client. Single method: get_json(). Handles connection errors and JSON parse errors separately. Logs every call with colored output.

🎯

Status

Wraps an HTTP status code. Truthy for 2xx only. Resolves human-readable name via HTTPStatus. Mixins: HasName, Validatable.

📦

Response

Immutable result from get_json(). Holds status, data, elapsed time, exception. Valid when 2xx and data exists. Mixin: Validatable.

Key Design Decisions

DecisionWhy
Boolean validity via Validatable Enables clean if response: checks at call sites
Separate error handling (connection vs JSON) Preserves status & elapsed even when JSON parsing fails
Static client (no instantiation) Simple utility — no state to manage
Status wraps HTTPStatus Adds validation layer + readable name without reinventing the enum
Response requires 2xx + data for validity Designed for JSON APIs where a body is always expected

10. Fun Facts

☕

418 I'm a Teapot

An April Fools' RFC from 1998 (RFC 2324) defined the "Hyper Text Coffee Pot Control Protocol". Google's teapot at google.com/teapot still returns 418.

🚀

HTTP/3 runs on UDP

After 30 years on TCP, HTTP/3 switched to QUIC (UDP-based). Connection setup dropped from 3 round-trips to 0–1.

💰

402 Payment Required

"Reserved for future use" since 1997. Nearly 30 years later, it's still not officially standardized. Some APIs use it anyway.

📈

~80% of all web traffic

HTTP carries roughly 80% of all internet traffic. The remaining 20% includes email (SMTP), DNS, gaming, and VoIP protocols.

📝

The first HTTP request

The first-ever webpage (info.cern.ch) was served in 1991 using HTTP/0.9 — a one-line protocol with no headers at all.

🔐

HTTPS overtook HTTP in 2017

Chrome started marking HTTP sites as "Not Secure" in 2017. By 2024, over 95% of Chrome page loads use HTTPS.
