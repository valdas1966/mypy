Client

f_http/client/

Purpose

  1. Static HTTP client that sends GET requests and returns parsed Response objects.
  2. Separates connection errors from JSON parse errors to preserve status and elapsed time.
  3. All methods are @staticmethod — no instance state needed.

Public API

Client

get_json

@staticmethod
def get_json(url: str,
             params: dict[str, Any] = None,
             headers: dict[str, str] = None) -> Response
  1. Sends an HTTP GET request to the given URL.
  2. Returns a Response with status, data, elapsed, and exception fields.
  3. On connection error: all fields are None except exception.
  4. On JSON parse error: preserves status and elapsed, sets data to None.
  5. On success: all four fields are populated.

Factory

Factory: type = None
  1. Class attribute wired to Factory in __init__.py.
  2. Provides valid() and invalid() for testing.

Usage Examples

Valid Request (200 OK)

from f_http.client import Client

response = Client.get_json(url="https://jsonplaceholder.typicode.com/posts/1")
assert response                    # True (status 200, data present)
assert response.status.code == 200
assert response.status.name == 'OK'
assert response.data               # parsed JSON dict
  1. Returns a truthy Response with 200 status and populated data.

Invalid Request (404 Not Found)

from f_http.client import Client

response = Client.get_json(url="https://jsonplaceholder.typicode.com/posts/999")
assert not response                 # False (status 404)
assert response.status.code == 404
assert response.status.name == 'NOT_FOUND'
assert not response.data            # empty dict for 404
  1. Returns a falsy Response with 404 status.

Using Factory

from f_http.client import Client

valid   = Client.Factory.valid()    # 200 response
invalid = Client.Factory.invalid()  # 404 response
  1. Factory provides pre-built test fixtures via jsonplaceholder API.
