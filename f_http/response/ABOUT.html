1. What is an HTTP Response?

When a client sends an HTTP request, the server replies with a response — a structured message containing everything the client needs: did it work? what data came back? how long did it take?

Anatomy of a Raw HTTP Response

HTTP/1.1 200 OK← status line
Content-Type: application/json
Content-Length: 247
X-Response-Time: 42ms
{"id": 1, "title": "Hello World"} ← body

The 4 Parts That Matter

1
Status
3-digit code + name. Did it succeed? Fail? Redirect?
2
Body
The actual payload — JSON, HTML, image, or nothing.
3
Headers
Metadata: content type, caching, cookies, auth tokens.
4
Timing
Round-trip time — how long the server took to answer.

4. Fun & Curious Facts

418
I'm a Teapot
RFC 2324 (1998) defined status 418 as an April Fools' joke for the "Hyper Text Coffee Pot Control Protocol." It stuck. Google's teapot page (google.com/teapot) returned 418 for years. In 2017, a campaign to remove it from real HTTP specs failed — the internet voted to keep the teapot.
0.6
The First HTTP Had No Status Codes
HTTP/0.9 (1991) had only one method (GET) and no status codes at all. The server simply returned the HTML document or closed the connection. Status codes arrived with HTTP/1.0 in 1996 — five years later.
451
Fahrenheit 451
Status 451 Unavailable For Legal Reasons (RFC 7725, 2016) was named after Ray Bradbury's novel about censorship. It's used when content is blocked by government order. The number was chosen as a direct literary reference.
204
The Invisible Success
204 No Content means "success, but I have nothing to say." It's the HTTP equivalent of a thumbs-up with no words. Used heavily for DELETE operations and tracking pixels — billions served daily, zero bytes of body.
301
The Most Expensive Redirect
When a site moves from http:// to https://, every single page returns 301 Moved Permanently. Google estimated that globally, 301 redirects add ~100ms per request. Across billions of daily requests, that's thousands of human-years of waiting — per year.
404
The Most Famous Error
404 Not Found is the only HTTP status code that became part of everyday language. Urban legend says it was named after room 404 at CERN where the first web servers lived — the room where requests "got lost." (The real origin is just the 4xx client-error numbering scheme, but the myth persists.)
∞
The Response That Never Ends
With Transfer-Encoding: chunked, a server can send a response that technically never finishes. Twitter's old streaming API did this — a single HTTP response that stayed open for hours, streaming tweets as chunks. One request, one response, millions of tweets.
