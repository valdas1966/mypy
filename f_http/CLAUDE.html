f_http

  1. Package path: f_http/

1. Purpose

  1. HTTP client infrastructure for sending GET requests and parsing responses.
  2. Four sub-modules: Client (request), Status (code wrapper), Response (result), URL (suffix extraction).
  3. All classes follow the ClassName.Factory pattern for test construction.

2. Public API

Client
  1. Static HTTP GET client that returns Response objects.
  2. Handles connection errors and JSON parse errors separately.
  3. Preserves status and elapsed even when JSON parsing fails.

get_json

@staticmethod
def get_json(url: str,
             params: dict[str, Any] = None,
             headers: dict[str, str] = None) -> Response
  1. Sends HTTP GET request to url with optional params and headers.
  2. Returns Response wrapping status, data, elapsed, and exception.
  3. Connection failure: all fields None except exception.
  4. JSON parse failure: status and elapsed preserved, data is None.
Status
  1. Thin wrapper around HTTPStatus with 2xx validation.
  2. Extends HasName and Validatable.

__init__

def __init__(self, code: int) -> None
  1. Wraps an HTTP status code; valid (truthy) only for 200–299.
  2. Resolves name via HTTPStatus; unknown codes get 'UNKNOWN'.

code

@property
def code(self) -> int
  1. Returns the raw HTTP status code.

name

@property
def name(self) -> str
  1. Human-readable name: 'OK', 'NOT_FOUND', 'UNKNOWN'.

__str__

def __str__(self) -> str
  1. Returns 'Code=200, Name=OK'.

__bool__

def __bool__(self) -> bool  # inherited from Validatable
  1. Returns True if code is 2xx.
Response
  1. Immutable result object returned by Client.get_json().
  2. Extends Validatable; truthy when status is 2xx and data exists.

URL
  1. Utility class for extracting file suffix from URL strings.
  2. Standalone; no inheritance from framework mixins.

  1. Wraps a URL string for metadata extraction.

suffix

def suffix(self) -> str | None
  1. Extracts file extension without leading dot.
  2. Tries path first (filename.ext), then mime_type/mimeType query param.
  3. Returns None if no suffix found.

3. Module Architecture

  1. Client.get_json() sends the HTTP request using the requests library.
  2. Client wraps the response code into a Status object.
  3. Client assembles Status, data, elapsed, and exception into a Response.
  4. Response delegates validity to Status truthiness plus data presence.
  5. URL is independent; no interaction with the other three classes.
Client ──uses──▶ requests (HTTP GET)
   │
   ├──creates──▶ Status(code)        → HasName + Validatable
   │
   └──creates──▶ Response(status, data, elapsed, exception) → Validatable
                    └── wraps Status

URL (standalone) → urllib.parse

5. Usage Examples

Fetch JSON and check result 
from f_http import Client

response = Client.get_json(url='https://api.example.com/posts/1')
if response:
    print(response.data)           # parsed JSON dict
    print(response.elapsed)        # e.g. 0.15
else:
    print(response.status.code)    # e.g. 404
    print(response.exception)      # error message or None
  1. Import Client from the top-level package.
  2. Call get_json() with a URL; optionally pass params and headers.
  3. Use boolean check on response: True means 2xx with data.
  4. Access .data, .status, .elapsed, .exception as needed.
Extract file suffix from URL 
from f_http import URL

url = URL(url='https://cdn.example.com/video.mp4')
assert url.suffix() == 'mp4'

# Falls back to mime_type query param
url = URL(url='https://cdn.example.com/stream?mime_type=video_mp4')
assert url.suffix() == 'mp4'
  1. Wrap any URL string in URL().
  2. Call .suffix() to get extension without dot.
  3. Returns None if no suffix is detectable.
