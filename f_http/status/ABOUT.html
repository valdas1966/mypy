HTTP Status Codes

What they are — and what f_http/status adds

1. What is an HTTP Status Code?

Every HTTP response carries a 3-digit integer that tells the client what happened.

1xx
Informational
Request received, processing continues.
2xx
Success
Request was received, understood, accepted.
3xx
Redirection
Further action needed to complete request.
4xx
Client Error
Bad syntax or request cannot be fulfilled.
5xx
Server Error
Server failed to fulfill a valid request.

Common Codes

2. The Problem with Raw Codes

# Raw integer — no context, no safety
status_code = response.status_code     # 404
if status_code == 200:                 # Magic number
    data = response.json()             # Might crash
ProblemConsequence
Just an intNo name, no context, no self-documentation
Manual range checksEasy to forget 201, 204, etc.
No truthinessCan't use if status: idiom
No type safetyAny integer passes — no validation

Usage

# Clean, self-documenting
from f_http.status import Status

status = Status(code=200)
if status:                              # True (2xx)
    print(status.name)                  # 'OK'
    print(status)                       # 'Code=200, Name=OK'

status = Status(code=404)
if not status:                          # True (not 2xx)
    print(status.name)                  # 'NOT_FOUND'
