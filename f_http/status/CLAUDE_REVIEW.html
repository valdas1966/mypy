  1. Bug: Factory.none() passes code=None, but Status.__init__ annotates code: int.
  2. The guard 200 <= code < 300 will raise TypeError if code is literally None in strict type checkers.
  3. At runtime CPython allows None <= 200 to be False in Python 2 but raises TypeError in Python 3.
  4. However, the code is not None short-circuit in and protects against the crash.
  5. Fix: Update type hint to code: int | None and code property return type to int | None.
