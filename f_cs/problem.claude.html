Overview

ProblemAlgo is the base class for defining algorithm input problems. It represents the "what needs to be solved" in the CS algorithm pattern.

Class Definition

from f_core.mixins.has.record import HasRecord

class ProblemAlgo(HasRecord):

A minimal base class that inherits HasRecord for serialization support and serves as a type constraint for Algo's Problem type parameter.

Usage Examples

Define a Problem

from f_cs.problem import ProblemAlgo

class SortingProblem(ProblemAlgo):
    RECORD_SPEC = {'size': lambda o: len(o.data)}

    def __init__(self, data: list[int]):
        super().__init__(name='SortingProblem')
        self.data = data

problem = SortingProblem(data=[5, 2, 8, 1, 9])

Use with Algo

from f_cs.algo import Algo

class SortingAlgo(Algo[SortingProblem, SortingSolution]):
    def _run(self) -> None:
        sorted_data = sorted(self.problem.data)
        self._output = SortingSolution(
            is_valid=True,
            stats=self._stats,
            result=sorted_data
        )

algo = SortingAlgo(problem=SortingProblem([5, 2, 8]))
solution = algo.run()
