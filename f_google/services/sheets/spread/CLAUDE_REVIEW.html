f_google/services/sheets/spread/

Summary

  1. Clean wrapper over gspread.Spreadsheet with minimal surface area.
  2. Correctly extends HasName to expose the spreadsheet title.
  3. Factory provides pre-configured credential shortcuts (OAuth and SA).
  4. The sheets property rebuilds Sheet objects on every call — minor performance consideration.
  5. Tests cover name, sheets listing, subscript access, and cell value retrieval.

1. Correctness

No change needed.

  1. Constructor correctly authorizes via gspread.authorize() and opens by key.
  2. __getitem__ correctly delegates to gspread.Spreadsheet.worksheet().
  3. Tests verify name, sheet count, sheet name, and cell value.

2. Readability / Naming

No change needed.

  1. Class name Spread is concise and consistent with the project abbreviation style.
  2. Parameter id_spread follows the project convention of prefixed identifiers.
  3. Factory method names (valdas, rami) match credential naming in f_google.creds.

No change needed.

Risk: Minor — each call triggers a network request to list worksheets. Fix: Cache the result and invalidate on mutation, or document the behavior. Not urgent for current usage patterns.

9. Usage & Usefulness

  1. How to use: Call Spread.Factory.valdas(id_spread=...) or Spread.Factory.rami(id_spread=...), then access sheets via spread['SheetName'] or spread.sheets.
  2. When useful: Any code that reads or writes Google Sheets data.
  3. Not overengineered: Thin wrapper — delegates to gspread for all Sheets API operations.
