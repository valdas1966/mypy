What's the Difference?

ask(prompt) → ResponseGemini
def ask(self, prompt: str) -> ResponseGemini:
    started = datetime.now()
    t_start = time.perf_counter()
    response = self._model.generate_content(prompt)
    elapsed = time.perf_counter() - t_start
    usage = response.usage_metadata
    return ResponseGemini(
        text=response.text,
        model=self._model_name,
        input_tokens=usage.prompt_token_count,
        output_tokens=usage.candidates_token_count,
        finish_reason=response.candidates[0]
                      .finish_reason.name,
        started=started,
        elapsed=elapsed
    )
ask_str(prompt) → str
def ask_str(self, prompt: str) -> str:
    response = self._model.generate_content(prompt)
    return response.text

✅

Use ask() Freely

The ResponseGemini wrapper adds zero measurable overhead. The ~11ms mean difference is within noise. You get tokens, timing, and finish_reason for free.

⚠️

Cold Start Penalty

The first call to ask() took 2.6s (vs 0.7s typical). This is likely Vertex AI session initialization, not the method itself — ask() ran first in the benchmark.

📊

Variance is Network

Both methods show similar variance (~0.2-0.25s stdev). The spread comes from network + Gemini server latency, not local Python code. P10-P90 ranges nearly overlap.

Recommendation

ask()
Default choice. Returns ResponseGemini with full metadata (tokens, timing, finish reason). No performance penalty.
ask_str()
Use when you only need the text and want the simplest possible interface. Saves ~5 lines of wrapper code, but gives up observability.
