Notes

  • A*-family = AStar, AStarLookup, AStarBPMX, AStarLookupBPMX, Dijkstra — identical frontier call pattern; differ only in priority shape and h-chain.
  • KDijkstra is single-pass observer. One inner Dijkstra pass; goal-pops detected via _early_exit. Dijkstra's priority (g, …) is goal-independent, so no refresh_priorities is ever needed — structurally KDijkstra matches A*-family (init only), NOT kA*-INC.
  • kA*-INC runs k sub-searches sharing one frontier. Between sub-searches: h changes (hi → hi+1), so refresh_priorities() drains (__iter__) → clear → re-pushes n× ⇒ O(n log n) per refresh. Same drain-and-rebuild pattern as kA*-AGG-eager._refresh_priorities.
  • kA*-INC goal re-push. Reached non-last goals are also re-pushed via algo._push(state=goal) so the next sub-search can expand them with the updated h (lazy re-push). One extra push per non-last goal reached.
  • Refresh naming is unified. AlgoSPP.refresh_priorities() (used by INC's inner AStar) and KAStarAgg._refresh_priorities() perform the same structural operation (drain + clear + re-push with fresh priorities); AGG adds bookkeeping (F-cache, events, opt-skip) but the core mechanism is identical.
  • kA*-AGG behaviour depends only on is_lazy: eager calls clear and __iter__ after every goal-find; lazy never does. Other params (is_opt, store_vector, agg) don't change frontier ops.
  • FrontierPriority __iter__ cost is dominated by QueueIndexed.to_iterable() = sorted(heap); FIFO iteration is plain O(n) deque traversal but is not invoked by any algorithm.
  • FIFO decrease is the inherited no-op from FrontierBase; never invoked by BFS/KBFS and never increments cnt_decrease.
  • push fall-through. On FrontierPriority, push on an existing state falls through to decrease_key internally — both paths are O(log n); the call-site counter still records cnt_push.
