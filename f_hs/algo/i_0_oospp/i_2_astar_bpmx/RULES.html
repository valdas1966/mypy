AStarBPMX — Rules

The two in-search mechanisms inside BPMXMixin: isolated pathmax (Felner Rules 1 / 2 / 3 at the immediate parent ↔ children neighborhood, depth 1 only) and BPMX(d) cascade (Rules 1 + 3 propagated outward through the d-level successor subtree, iterated to a fixed point). Faithful to Felner et al., Inconsistent Heuristics in Theory and Practice, AIJ 2011.

Orientation — two orthogonal mechanisms

rule_bpmx ∈ {'1', '2', '3'}

Isolated rule fired once per expansion at the immediate parent → children neighborhood (depth 1 only). No cascading. Pick exactly one rule, or None to disable.

rule_bpmx = 'CASCADE'

BPMX(d) cascade — alternating Rule 3 (bottom-up) and Rule 1 (top-down) sweeps over a d-level BFS subtree, iterated until no pass tightens any node. None ⇒ propagate through the full reachable subtree (visited-set bounded).

Storage. Lifted h-values persist via an HBounded layer in the heuristic chain. Admissibility is preserved by construction (triangle inequality). Lifts on states already in the frontier with stale priorities are left as-is — A* with inconsistent heuristics accepts stale priorities (Martelli bound; Felner §4).

Rule 1 Mero (1984) — parent → child

h′(c) = max(h(c),  h(p) − w(p, c))

For every child c of the popped parent p, lift c's h from the parent's h minus the edge weight. Each child is potentially lifted; the parent is unchanged.

Rule 2 Felner — children → parent via min

h′(p) = max(h(p),  mini(h(ci) + w(p, ci)))

The parent's h is bounded from below by the cheapest available child path: take the child whose h(c) + w(p, c) is smallest, and use that as a lower-bound estimate on the parent's distance to goal. Only the parent is lifted; children are unchanged.

Rule 3 Felner — single child → parent (reverse pathmax)

h′(p) = max(h(p),  maxc(h(c) − w(c, p)))

The parent's h is lifted from its strongest child — the one whose h(c) − w(c, p) is largest. Reverse-pathmax: if some neighbour estimates a high distance to goal, the parent must be at least that distance minus the edge cost back. Only the parent is lifted.

BPMX(d) Cascade — Felner Algorithm 2

BFS-spans the d-level successor subtree from the popped state, then alternately applies Rule 3 bottom-up (parents lift from strongest children) and Rule 1 top-down (parents push to children), iterated until no pass tightens any node. depth=None ⇒ full reachable subtree, visited-set bounded.

Per-iteration

  1. Bottom-up Rule 3 sweep over levels d−1 → 0.
  2. Top-down Rule 1 sweep over levels 0 → d−1.
  3. If neither sweep tightened any state ⇒ converged.

Cached descendants

States with is_perfect True (HCached layer) are skipped from lift mutation — cached h is already the optimal h*. The cascade still traverses through them but never writes new bounds.

Single-axis API: rule_bpmx picks one of the four mechanisms (or None for off); depth_bpmx controls how far the BFS subtree walks. Rule 2 is structurally depth-1 only — its operator consumes a parent + its full children set, with no chained- grandparent analogue. Constructor enforces depth_bpmx == 1 for Rule 2.
