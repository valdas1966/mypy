Reading the matrix

5 counters omitted (all kept in the class scaffold). cnt_decrease ≡ 0 (no priority-decrease on this fixture); given that, cnt_push ≡ cnt_generated and cnt_pop ≡ cnt_expanded + 3 — affine duplicates. cnt_h_search dropped by request: it is a pure heuristic-evaluation cost counter.
cnt_cache_hits_at_init — different reason. It is the algorithm's headline metric, not vestigial. It is constant (= 1) here only because every config carries the cache and the S1→T optimal path geometry is param- independent — exactly S2 lands on it. The value is fixture-geometry invariant, not structural: across fixtures it ranges [0, k−1] and is the sole counter-side (recording-OFF) witness of the cache-hit-at-init win. Shown via the fixture callout and pinned by study/oracle.py / _tester_counters.py — just not as a flat heatmap column here.
cnt_prop_waves is MAX, not SUM. It is the deepest wave ladder any single sub-search ran (a propagation-depth horizon), not the total across the k sub-searches — mirroring how mem_* is peak-aggregated. So depth_1/2/3 read 1/2/3 (the cap that bound) and depth_inf reads 5 (the deeper of the two converging sub-searches; the first sub-search propagates 0 — empty carried cache). Wave-work is carried by the summed cnt_prop_attempts / cnt_prop_lifts.
cnt_bpmx_depth is MAX, not SUM. Same horizon logic: the deepest BFS-level a lift fired in any single sub-search — capped by where lifts actually stop, not the depth_bpmx param (e.g. depth_bpmx=∞ still reads the true reached depth). The inner mixin already tracks it per-search as a max; the orchestrator MAX-aggregates across sub-searches (it does not sum k maxima). Lift-work is carried by the summed cnt_bpmx_attempts / cnt_bpmx_lifts (formerly cnt_bpmx_successes).
Consequence — 6 duplicate rows. cnt_h_search was the sole separator of two BPMX depth tails, so on the shown counters only 16 of 22 configs differ. Collapsed configs are tagged ≡ canonical and dimmed — depth there changes nothing but heuristic-call cost. Full 22-way distinctness (incl. cnt_h_search / push / pop) is verified by study/oracle.py and _tester_counters.py.
