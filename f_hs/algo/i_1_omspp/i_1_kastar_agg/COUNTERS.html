Invariant = 14. Same successor function + same start + same goals → same set of unique states first- encountered. Counts every _handle_child first-encounter branch — equal across all 8 configs.
  • 3 counters are invariant across all configs: cnt_decrease=0, cnt_expanded=9, and cnt_generated=14. They reflect the search-graph structure under the INC-symmetric goal-handling, untouched by any axis. Under Path D (2026-05-11) cnt_h_search and cnt_phi_search are no longer invariant — lazy modes count their pop-time staleness h / Φ work here too, making the lazy values larger than the eager values for the same store_vector.
  • Lazy mode invariant: cnt_h_update = cnt_phi_update = 0. Lazy never enters PHASE_UPDATE; its active-set- change response is folded into the search loop via pop-time staleness checks. The counter axis mirrors the structural phase axis exactly.
  • store_vector wins on cnt_h_search AND cnt_h_update (drives later h-reads to the cached vector → cnt_h_search falls to its first-encounter floor of 34; cnt_h_update falls to 0).
  • is_opt wins on cnt_phi_update / cnt_h_update (eager refresh recompute skipped when responsible goal still active) AND on cnt_phi_search / cnt_h_search in lazy (responsible-goal short-circuit skips most pop-time recomputes).
  • Lazy goal-repush skip (2026-05-11): at goal- find, lazy mode pushes the goal back with its stale F (= popped F) and lets the lazy stale-pop machinery refresh it on the next pop. Cost: +1 stale pop per non-last goal (here +2 → cnt_pop 16 → 18, cnt_push 20 → 22; stale subset derivable as cnt_pop − cnt_expanded − #on_goal goes 4 → 6). Benefit: lazy mode does literally no active-set-change-response work between sub-search segments.
  • Goal-handling is INC-symmetric: at each goal pop, emit on_goal then push (re-push, if non-last). The just-found goal is NOT force-expanded; its successors are reached via other paths during the search. The last active goal is never re-pushed.
  • No single config wins everything — eager minimizes pops, lazy minimizes pushes; opt+sv layer cheaply on top of either.
