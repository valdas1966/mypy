Early-Stop / Mission Cancellation
run k=200, cancel mid-flight, price what each algorithm already spent

Launch a full k=200 OMSPP search, then let an external event cancel it at a progress milestone — r of the k objectives completed. At each cancellation point measure the cost already incurred: expanded nodes, peak memory, runtime. Compare kA*-INC vs kA*-AGG. Models real mission abort, not "find the nearest goal."

Bottom line Both algorithms solve the same full k=200 problem. We freeze each at the moment its r-th objective is completed and read off the cost it had paid. Because the trajectory to the r-th goal is a prefix of the full run, one continuing run per algorithm, snapshotted at each milestone, yields all five r — exactly for expanded-nodes and peak-memory, faithfully for runtime. Headline: how INC's incremental 1-h/node compares to AGG's k-vector tax as a function of when the mission is cancelled.

1Framing — mission cancellation, not nearest-goal

Same problem, frozen at r objectives done This is not "reach the nearest goal and stop" (that would make goal ordering a confound). Both algorithms run the identical k=200 search and make progress toward all goals; an external abort freezes them at "r of k completed." Fixed-order INC having finished t₁…t_r while AGG-MIN finished its r nearest is each algorithm's real behavior — exactly the incurred cost we price. So fixed order is correct here; no ordering knob.
