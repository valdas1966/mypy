0What I think of the 2-parameter design

Verdict — adopt, with guardrails The two axes are orthogonal and individually well-motivated, and they amplify the gap through different mechanisms — so the cross-join is not redundant: it measures interaction, which is the scientifically interesting part and the kind of result top venues reward. The 1-D max_steps sweep is simply one row of this grid.

✅ Why it's worth doing

  • Two mechanisms, one figure. min_dist grows the shared trunk (more all-goals-active nodes before any goal is found); max_steps grows the dispersion (goals stay active longer inside the region). Both raise the R factor — but independently.
  • Attribution. The grid separates "INC wins because goals are far" from "INC wins because goals are scattered" — a stronger claim than either alone.
  • Phase diagram. A heatmap of AGG/INC ratio over (distance × spread) is the natural headline figure for direction #4.
  • Cheap reuse. Rides the existing s_0…s_8 pipeline almost verbatim.

⚠ Four things to fix first

  • Degenerate corner. When min_dist ≲ max_steps the goal diamond reaches back to (or past) the start → start becomes a near/actual goal. Mask those cells.
  • min_dist is a floor today. Parent s_2 accepts dist ≥ min_dist, so values pile just above it. For a clean axis, sample into bands [d±δ].
  • Map-size thinning. min_dist=500 + radius 100 needs map diameter ≳ 700; small maps drop out → unbalanced panel biased to big maps.
  • Cost & pairing. 25 cells; the far/large cells dominate runtime. And min_dist is a between-pair factor (can't pair across it), unlike max_steps.
Predicted interaction (the science) Gap ≈ R × F, F fixed (grid+Manhattan). R rises along both axes, so the ratio is maximal at far + scattered (bottom-right of the grid). Open question the grid answers: is the effect additive (independent contributions), multiplicative (trunk × dispersion compound), or saturating (one axis caps the other)?

Because the trunk is expanded by both algorithms identically, min_dist mainly widens the per-node cost gap (h-calls → runtime, k-vector → memory), not the expanded-node gap. That asymmetry is itself a clean, testable prediction the grid surfaces.

The 5×5 grid — degenerate corner & map thinning

The goal diamond has radius = max_steps; the start sits min_dist from its center. When min_dist ≤ max_steps the start is inside the region; when only slightly larger, the nearest goals crowd it. Mask rule: keep a cell only if min_dist ≥ max_steps + margin (e.g. margin = 50 ⇒ nearest goal ≥ ~50 from start).

Mixed pairing — state it in the analysis max_steps is within-pair → analyze paired log-ratios (variance cancels along this axis). min_dist is between-pair → cluster-bootstrap over maps within each band. The experimental unit is (map, min_dist-band, pair) → a max_steps series. Bind sample size by the expanded-nodes ratio (smallest effect).
