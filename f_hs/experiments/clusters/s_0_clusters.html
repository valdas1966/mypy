s_0_clusters.py

f_hs/experiments/clusters/ Sample N random ClusterDiamonds per GridMap and stream metadata to a CSV on Google Drive.

Flow

*.map files
→
Load GridMap
per map
→
Sample N
ClusterDiamond.Factory.random
→
Drive
one .csv

Parameters

argmeaning
grids list[GridMap] to sample on
path_drive_csv output path on Drive
steps Manhattan radius of each diamond
min_cells minimum valid cells per cluster
n samples per grid
max_tries retries per sample (default 100)

CSV columns

columnsource
domain grid.domain
map grid.name
center_row cluster.center.row
center_col cluster.center.col
steps cluster.steps
cells len(cluster) — valid cells in BFS ball

When to use  ·  when not to

✓ Use

  • Characterise the distribution of cluster size for a fixed (steps, min_cells).
  • Profile where random centers land on each map.
  • Sanity-check parameter choices before running the pair-cluster pipeline.

✗ Do not use

  • For pair experiments — use s_1_pair_clusters.py.
  • For one-off calls — invoke ClusterDiamond.Factory.random(…) directly.
  • If you need full ClusterDiamond objects later — CSV keeps metadata only.
