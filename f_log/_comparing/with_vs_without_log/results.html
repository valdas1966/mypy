Key Findings

  • SAFE Disabled vs No-Log: ~8x overhead. Having log.debug() calls with logging disabled adds negligible cost.
  • MODERATE Console vs Disabled: ~77x overhead. Active console logging adds significant cost due to string formatting and stream I/O.
  • HEAVY File vs Console: ~22x overhead. File logging with rotation is the most expensive due to synchronous disk writes.
  • TOTAL File vs No-Log: ~13,170x total overhead. File logging is orders of magnitude slower than code without any logging.

Recommendations

1. Keep your log statements in the code. With setup(enabled=False), the overhead is minimal (~8x) and not worth stripping calls from hot paths.

2. Use DEBUG level wisely. In production, set level to WARNING or higher. Messages below the level threshold are skipped before formatting — near-zero cost.

3. Prefer console over file when possible. Console sink is ~22x faster than file sink. Pipe stdout to a file externally if you need persistence.

4. For high-throughput loops, disable logging or guard with if log.isEnabledFor(DEBUG) to avoid even the function-call overhead.
